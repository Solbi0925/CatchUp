# AGENTS.md

## Product

Catchup은 대학생이 강의계획서, LMS 공지, 과제, 시험, 개인 일정을 한곳에서 정리하고 앞으로 4주간의 일정을 참고해 오늘/이번 주에 무엇부터 해야 할지 판단하도록 돕는 AI 학업 일정 매니저 MVP이다.

## Source of truth

작업 전에는 반드시 아래 문서를 순서대로 확인한다.

1. `PRD.md`: MVP 범위, 기능, 제외 기능, 성공 기준
2. `README.md`: 프로젝트 소개, 팀 정보, 현재 문서 구조

문서와 다른 요구가 생기면 먼저 `PRD.md`를 기준으로 판단한다.

## Build and verify

현재 단계는 기획 문서 작성 단계이다.

- 새 작업 전 `PRD.md`와 `README.md`의 내용이 서로 충돌하지 않는지 확인한다.
- 새 문서를 만들면 제목, 목적, 포함 범위가 명확한지 확인한다.
- 기능을 제안할 때는 10주 MVP 안에서 구현 가능한지 확인한다.
- 코드가 생긴 뒤에는 실행 방법, 테스트 방법, 환경 변수 사용 여부를 문서에 남긴다.

## Guardrails

- 실제 학생 이름, 학번, 전화번호, 이메일, 성적, 시간표, 캘린더 등 개인정보를 사용하지 않는다.
- 테스트에는 가짜 학생 데이터와 샘플 일정만 사용한다.
- 실제 학교 LMS 계정, 학교 포털 계정, Google 계정의 민감한 정보를 저장하지 않는다.
- API 키, OAuth 토큰, 비밀번호는 문서나 코드에 직접 적지 않는다.
- 업로드 예시는 실제 학생 자료가 아니라 익명화된 샘플 자료만 사용한다.
- AI 추천은 참고용 계획으로 표현하고, 정확한 완료 확률이나 성과 예측처럼 보이게 쓰지 않는다.

## Scope control

- 모든 작업은 `PRD.md`의 1차 MVP 범위 안에서 진행한다.
- 자동 LMS 완전 연동, 완벽한 OCR, 장기 학습 성과 예측, 친구 공동 일정, 모바일 앱 정식 출시는 만들지 않는다.
- 월간 상세 계획은 만들지 않고, 매주 계획 생성 시 앞으로 4주간의 마감일과 시험일만 참고한다.
- AI 챗봇 수정 요청은 비용 관리를 위해 횟수 제한이 있는 기능으로 다룬다.
- 매일 미완료 할 일을 기준으로 AI가 전체 계획을 자동 재생성하는 기능은 낮은 우선순위로 둔다.
- 새 기능을 추가하고 싶으면 먼저 “MVP에 꼭 필요한가?”를 확인한다.
- 기능이 커지면 이번 MVP가 아니라 이후 확장 기능으로 분리한다.

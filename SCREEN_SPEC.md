# CatchUp Screen Spec

## 1. 문서 목적

이 문서는 `IA.md`와 `USER_FLOW.md`를 바탕으로 CatchUp MVP의 화면별 구성 요소, 표시 정보, 사용자 행동, 상태를 정의한다.

현재 화면 구조는 아래를 기준으로 한다.

```text
온보딩
-> Google Calendar 연동
-> 메인 앱

하단 네비게이션
├─ 오늘의 할일
├─ 주간,월간 일정
└─ 자료 업로드

전역 플로팅
└─ AI 챗봇
```

## 2. 공통 화면 규칙

### 네비게이션

- 메인 앱은 하단 네비게이션 바를 사용한다.
- 하단 네비게이션에는 `오늘의 할일`, `주간,월간 일정`, `자료 업로드` 3개 탭만 둔다.
- AI 챗봇은 하단 네비게이션의 개별 탭으로 만들지 않는다.
- AI 챗봇은 모든 메인 화면에서 접근 가능한 전역 플로팅 버튼으로 제공한다.
- 이전의 가로 스크롤 페이지 이동은 사용하지 않는다.

### 공통 데이터 표시 원칙

- 학업 일정과 개인 일정은 출처를 구분해 표시한다.
- AI가 추출한 정보 중 불확실한 값은 `확인 필요`로 표시한다.
- 추천 근거는 짧게 먼저 보여주고, 필요 시 플로팅 AI 챗봇에서 자세히 질문할 수 있다.
- 수치형 완료 확률은 표시하지 않는다.
- 오늘 할일과 주간 계획 항목의 직접 수정 UI는 초기 구현에서 제공하지 않는다.

### 공통 제한

- 주간 계획 생성은 사용자가 설정한 요일과 시간에 주 1회만 가능하다.
- 한 주의 계획 범위는 월요일부터 일요일까지다.
- AI 챗봇을 통한 일일/주간 계획 조정 요청은 1주에 최대 3번만 가능하다.
- Google Calendar 외 다른 외부 캘린더는 연결하지 않는다.

### 공통 상태

| 상태 | 설명 |
| --- | --- |
| 빈 상태 | 아직 업로드, 연결, 계획 생성이 완료되지 않은 상태 |
| 로딩 상태 | 자료 추출, 일정 불러오기, 계획 생성 중인 상태 |
| 확인 필요 | AI 추출 결과 중 사용자가 검토해야 하는 항목이 있는 상태 |
| 오류 상태 | 업로드 실패, 추출 실패, 캘린더 연결 실패 등 |
| 정상 상태 | 일정과 계획이 표시되는 상태 |
| 제한 도달 | 주간 계획 생성 또는 AI 챗봇 조정 요청 가능 횟수를 모두 사용한 상태 |

## 3. 온보딩 화면

### 목적

사용자가 CatchUp의 핵심 가치와 사용 흐름을 빠르게 이해하고 시작하도록 한다.

### 화면 구성

```text
[앱 이름: CatchUp]
[한 줄 소개]
[핵심 기능 요약]
[시작 버튼]
```

### 표시 정보

- 앱 이름
- 학업 자료와 Google Calendar를 바탕으로 오늘과 이번 주에 해야 할 일을 추천한다는 설명
- 핵심 기능
  - 학업 자료 업로드
  - AI 일정 추출
  - Google Calendar 기반 주간 계획 생성
  - 오늘의 할일 확인
  - 플로팅 AI 챗봇 기반 계획 생성 및 조정

### 사용자 행동

- 앱 소개 확인
- 시작 버튼 선택
- Google Calendar 연동 화면으로 이동

## 4. Google Calendar 연동 화면

### 목적

사용자의 개인 일정을 불러와 학업 일정과 함께 주간 계획 생성에 반영할 수 있도록 한다.

### 화면 구성

```text
[Google Calendar 연동 안내]
[연동 필요 이유]
[Google Calendar 연결 버튼]
[연결 상태]
```

### 표시 정보

- Google Calendar 연동 안내
- 개인 일정과 학업 일정 충돌 방지 설명
- 주간 계획 생성 시 개인 일정이 반영된다는 설명
- 연결 상태
  - 연결 전
  - 연결 중
  - 연결 완료
  - 연결 실패

### 사용자 행동

- Google Calendar 연결 버튼 선택
- 권한 승인
- 연결 실패 시 재시도
- 연결 완료 후 메인 앱 진입

### 오류 상태

- 연결 실패 시 재시도 안내를 표시한다.
- Google Calendar 연결이 완료되지 않으면 개인 일정은 계획 생성에 반영하지 않는다.

## 5. 메인 공통 레이아웃

### 화면 구성

```text
[현재 탭 콘텐츠]

[플로팅 AI 챗봇 버튼]

[하단 네비게이션]
- 오늘의 할일
- 주간,월간 일정
- 자료 업로드
```

### 플로팅 AI 챗봇 버튼

- 모든 메인 탭에서 표시한다.
- 화면 오른쪽 하단 또는 하단 네비게이션 위에 배치한다.
- 사용자가 누르면 AI 챗봇 패널을 연다.
- 버튼은 주요 콘텐츠를 가리지 않도록 배치한다.

## 6. 탭 1: 오늘의 할일

### 목적

AI가 이번 주 계획에서 오늘로 배치한 학습 항목과 오늘 예정 일정을 빠르게 확인한다.

### 화면 구성

```text
[오늘 날짜]

[오늘의 할일]

[오늘 예정 일정 요약]

[플로팅 AI 챗봇 버튼]
```

### 섹션 1: 오늘 날짜

#### 표시 정보

- 날짜
- 요일
- 오늘 할일 개수
- 오늘 예정 일정 개수

### 섹션 2: 오늘의 할일

#### 정의

오늘의 할일은 AI가 앞으로 4주간의 일정, 난이도, 마감일, 우선순위, 예상 소요시간, Google Calendar 개인 일정을 고려해 주간 계획에서 오늘로 배치한 학습 항목이다.

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 과목명 또는 자료 출처 | 어떤 수업 또는 자료에서 나온 일인지 표시 |
| 할일 제목 | 오늘 해야 할 학습 행동 |
| 예상 소요시간 | 계획 이해를 돕는 시간 정보 |
| 우선순위 | 높음, 보통, 낮음 |
| 완료 여부 | 완료 체크 상태 |
| 추천 근거 요약 | 왜 오늘 해야 하는지 한 문장으로 표시 |

#### 할일 유형

- 과제 진행
- 시험 공부
- 수업 준비
- 복습 또는 예습

#### 사용자 행동

- 오늘의 할일 확인
- 완료 체크
- 추천 근거 확인
- 계획 변경이 필요하면 플로팅 AI 챗봇 열기

#### 빈 상태

- 주간 계획이 없으면 `아직 오늘의 할일이 없습니다.`를 표시한다.
- 플로팅 AI 챗봇에서 주간 계획 생성을 요청하도록 안내한다.

### 섹션 3: 오늘 예정 일정 요약

#### 정의

오늘 예정 일정은 날짜 기준으로 오늘 예정된 일정이다. AI가 임의로 배치한 학습 항목이 아니라, 학업 자료에서 추출된 일정 또는 Google Calendar 개인 일정이다.

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 일정명 | 마감, 제출, 시험, 개인 일정 제목 |
| 시간 | 오늘 중 예정 시간 또는 종일 |
| 유형 | 마감, 제출, 시험, 중요 공지, 개인 일정 |
| 출처 | 업로드 자료 또는 Google Calendar |

#### 사용자 행동

- 오늘 예정 일정 확인
- 일정 출처 확인

### 탭 1에서 제외하는 기능

- 주간 계획 생성 입력
- 계획 조정 입력
- 학업 자료 업로드
- 오늘 할일의 시간, 우선순위, 내용 직접 수정

## 7. 탭 2: 주간,월간 일정

### 목적

학업 일정과 Google Calendar 개인 일정을 월간 캘린더로 확인하고, AI가 생성한 이번 주 계획을 주간 플래너로 확인한다.

### 화면 구성

```text
[월간 일정 요약 캘린더]

[주간 플래너]

[플로팅 AI 챗봇 버튼]
```

### 섹션 1: 월간 일정 요약 캘린더

#### 역할

월간 단위로 학업 일정과 개인 일정을 확인한다. 월간 화면은 일정 확인용이며, 월간 세부 자동 계획을 생성하지 않는다.

#### 표시 정보

- AI가 학업 자료에서 추출한 마감일
- 제출일
- 시험일
- 중요 공지 일정
- Google Calendar 개인 일정

#### 표시 규칙

- 날짜는 일반 달력처럼 숫자로 표시한다.
- 일정이 있는 날짜는 날짜 숫자 아래에 점으로 일정 개수를 표시한다.
- 일정 1개는 점 1개, 일정 2개는 점 2개, 일정 3개는 점 3개로 표시한다.
- 점은 일정의 상세 내용을 직접 보여주지 않고, 해당 날짜에 일정이 있다는 신호로만 사용한다.
- 초기 구현에서는 점 색상으로 학업 일정과 개인 일정을 구분하지 않는다.
- 초기 구현에서는 확인 필요 일정을 일반 일정과 다르게 표시하지 않는다.

#### 사용자 행동

- 날짜 선택
- 선택한 날짜의 일정 상세 팝업 확인
- 일정 출처 확인

### 섹션 2: 일정 상세 팝업

날짜를 선택하면 해당 날짜의 일정 상세 요약을 팝업으로 표시한다.

```text
7일
- 알고리즘 과제 제출 마감일
- Google Calendar 개인 일정
```

#### 팝업 표시 정보

| 항목 | 설명 |
| --- | --- |
| 선택 날짜 | 사용자가 선택한 날짜 |
| 일정명 | 해당 날짜의 일정 제목 |
| 유형 | 마감, 제출, 시험, 중요 공지, 개인 일정 |
| 출처 | 업로드 자료 또는 Google Calendar |

### 섹션 3: 주간 플래너

#### 역할

AI가 생성한 이번 주 계획을 월요일부터 일요일까지 요일별로 보여준다.

#### 표시 정보

- 요일
- 날짜
- 각 요일의 학습 항목
- 예상 소요시간
- 우선순위
- 완료 여부
- 개인 일정과 학습 일정의 충돌 여부
- 미완료 항목의 재배치 결과
- 추천 근거 요약

#### 사용자 행동

- 요일별 계획 확인
- 완료 체크
- 추천 근거 확인
- 계획 변경이 필요하면 플로팅 AI 챗봇 열기

#### 빈 상태

- 주간 계획이 없으면 `아직 이번 주 계획이 없습니다.`를 표시한다.
- 플로팅 AI 챗봇에서 주간 계획 생성을 요청하도록 안내한다.

### 탭 2에서 제외하는 기능

- 주간 계획 생성 입력
- 계획 조정 입력
- 학업 자료 업로드
- 주간 계획 항목의 시간, 날짜, 우선순위, 내용 직접 수정
- 전체 월간 세부 계획 생성

## 8. 탭 3: 자료 업로드

### 목적

학업 자료를 업로드하고 AI가 추출한 정보를 직접 확인 및 수정한다. 업로드 자료와 수정된 추출 정보는 플로팅 AI 챗봇이 주간 계획을 생성할 때 사용하는 핵심 근거가 된다.

### 화면 구성

```text
[학업 자료 업로드]

[업로드한 자료 목록]

[AI 추출 결과 확인 및 수정]

[플로팅 AI 챗봇 버튼]
```

### 섹션 1: 학업 자료 업로드

#### 표시 정보

- 업로드 영역
- 지원 파일 형식 안내
- 업로드 진행 상태

#### 지원 범위

- MVP에서는 PDF 또는 이미지 중 하나의 파일 형식만 지원한다.
- 모든 파일 형식을 동시에 지원하지 않는다.

#### 사용자 행동

- 강의계획서 업로드
- LMS 공지 업로드
- 과제 명세서 업로드
- 업로드 실패 시 재시도

### 섹션 2: 업로드한 자료 목록

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 파일명 | 업로드한 학업 자료 이름 |
| 업로드 날짜 | 파일을 올린 날짜 |
| 추출 상태 | 추출 중, 추출 완료, 확인 필요, 추출 실패 |
| 확인 필요 여부 | 사용자가 검토해야 하는지 표시 |

#### 사용자 행동

- 업로드한 파일 확인
- 추출 상태 확인
- 확인 필요 파일 선택

### 섹션 3: AI 추출 결과 확인 및 수정

#### 역할

AI가 학업 자료에서 추출한 정보를 사용자에게 보여주고, 사용자가 틀린 부분을 직접 입력해 수정한다.

#### 표시 정보

| 필드 | 설명 |
| --- | --- |
| 과제명 | 추출된 과제 이름 |
| 마감일 | 과제 또는 제출 마감일 |
| 시험일 | 시험 예정일 |
| 제출 방식 | 온라인 제출, LMS 제출 등 |
| 준비물 | 수업 또는 과제 준비물 |
| 중요 공지 | 놓치면 안 되는 공지 |
| 수업 일정 정보 | 수업 관련 일정 |

#### 상태

| 상태 | 설명 |
| --- | --- |
| 추출 중 | AI가 정보를 추출하는 중 |
| 추출 완료 | 사용자가 확인할 수 있는 상태 |
| 확인 필요 | 일부 값이 불확실하거나 누락된 상태 |
| 저장 완료 | 사용자가 수정한 정보가 반영된 상태 |

#### 사용자 행동

- 추출 결과 확인
- 틀린 값 직접 수정
- 누락된 값 직접 입력
- 확인 필요 항목 검토
- 수정 내용 저장
- 플로팅 AI 챗봇을 열어 업로드 자료 기반 주간 계획 생성 요청

#### 제외 사항

- AI 챗봇이 추출 정보를 대신 수정하는 흐름은 포함하지 않는다.
- 추출 결과 수정은 사용자의 직접 입력으로 처리한다.

## 9. 전역 플로팅 AI 챗봇

### 목적

사용자가 어떤 메인 화면에 있든 AI 챗봇을 열어 주간 계획 생성, 계획수립 요청사항 입력, 생성된 계획 조정, 추천 근거 질문을 할 수 있도록 한다.

### 화면 구성

```text
[플로팅 AI 챗봇 버튼]

버튼 선택 시:
[AI 챗봇 패널]
- 대화 영역
- 자연어 입력 영역
- 주간 계획 생성 설정 요약
- 주간 계획 생성 상태
- 계획 조정 잔여 횟수
```

### 플로팅 버튼

#### 표시 규칙

- 모든 메인 탭에서 표시한다.
- 하단 네비게이션 위 또는 화면 오른쪽 하단에 배치한다.
- 현재 콘텐츠와 하단 네비게이션을 가리지 않도록 배치한다.

#### 사용자 행동

- 버튼 선택
- AI 챗봇 패널 열기
- 패널 닫기

### AI 챗봇 패널

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 대화 영역 | AI와 사용자의 메시지 표시 |
| 자연어 입력 영역 | 계획 생성 요청, 계획 조정 요청, 질문 입력 |
| 주간 계획 생성 요일 | 사용자가 설정한 생성 요일 |
| 주간 계획 생성 시간 | 사용자가 설정한 생성 시간 |
| 다음 생성 예정 시각 | 다음 주간 계획 생성 가능 시각 |
| 이번 주 생성 여부 | 이번 주 계획 생성 완료 여부 |
| 조정 잔여 횟수 | 이번 주 AI 챗봇 계획 조정 가능 횟수 |

#### AI가 참고하는 정보

- 자료 업로드 화면에 업로드된 학업 자료
- 사용자가 직접 수정한 AI 추출 결과
- Google Calendar 개인 일정
- 완료한 일과 미완료 항목
- 사용자의 자연어 요청사항

#### 입력 가능 요청

- 주간 계획 생성 요청
- 계획수립 요청사항 입력
- 오늘 계획 조정 요청
- 이번 주 계획 조정 요청
- 추천 근거 질문

#### 요청 예시

- "이번 주 계획 짜줘."
- "자료 업로드 화면에 올린 강의계획서랑 과제 명세서 기반으로 이번 주 계획 짜줘."
- "일요일에는 쉬는 시간을 많이 확보해줘."
- "오늘 할 일을 조금 줄여줘."
- "수요일에 개인 일정이 많으니 그날 할 일을 줄여줘."
- "왜 이 과제를 오늘 해야 해?"

#### 응답에 포함할 정보

- 생성 또는 조정된 계획 요약
- 반영한 사용자 요청사항
- 추천 근거
- 충돌 또는 위험도 안내
- 이번 주 남은 계획 조정 횟수

### 제한 상태

| 상태 | 화면 동작 |
| --- | --- |
| 주간 계획 생성 전 | 설정된 요일과 시간이 되면 주간 계획 생성 가능 |
| 이번 주 생성 완료 | 같은 주 안의 추가 주간 계획 생성 요청 차단 |
| 조정 잔여 1~3회 | AI 챗봇 계획 조정 요청 가능 |
| 조정 잔여 0회 | 추가 조정 요청 차단, 다음 주 재사용 가능 안내 |

### 표현 제한

- 정확한 완료 확률 수치는 표시하지 않는다.
- `우선순위 높음`, `완료 위험도 높음`처럼 정성적 표현을 사용한다.

## 10. 주요 데이터 표시 구분

### 오늘의 할일과 오늘 예정 일정 구분

| 구분 | 오늘의 할일 | 오늘 예정 일정 |
| --- | --- | --- |
| 기준 | AI가 오늘로 배치한 학습 행동 | 날짜 기준으로 오늘 예정된 일정 |
| 생성 방식 | 주간 계획 생성 결과 | 학업 자료 추출 또는 Google Calendar |
| 예시 | 과제 2시간 진행, 시험 공부 40분 | 오늘 과제 마감, 오늘 시험, 개인 약속 |
| 사용자 행동 | 완료 체크, AI 챗봇에 계획 조정 요청 | 일정 확인, 출처 확인 |

### 학업 일정과 개인 일정 구분

| 구분 | 학업 일정 | 개인 일정 |
| --- | --- | --- |
| 출처 | 업로드한 학업 자료 | Google Calendar |
| 예시 | 마감일, 제출일, 시험일, 중요 공지 | 병원, 약속, 아르바이트, 개인 일정 |
| 수정 위치 | 자료 업로드 탭의 AI 추출 결과 수정 영역 | Google Calendar 원본 또는 연결 상태에 따라 반영 |

## 11. MVP 제외 화면 및 기능

- AI 챗봇을 하단 네비게이션의 개별 탭으로 제공하는 화면
- 네 번째 메인 탭
- 별도 월간 상세 계획 페이지
- 모든 LMS 자동 연동 화면
- Google Calendar 외 캘린더 연결 화면
- 친구와 공동 일정 관리 화면
- 장학금, 근로, 대외활동 추천 화면
- 정확한 완료 확률을 보여주는 예측 화면
- 모든 파일 형식을 동시에 지원하는 업로드 화면
- 오늘 할일 또는 주간 계획 항목을 사용자가 직접 수정하는 화면
- 같은 주 안에서 주간 계획을 여러 번 생성하는 화면
- AI 챗봇 계획 조정 요청을 1주에 3번 넘게 허용하는 화면

## 12. 초기 구현 이후 추가 검토 기능

- 사용자가 오늘 할일 또는 주간 계획 항목의 시간, 날짜, 우선순위, 내용을 직접 수정하는 기능
- 일정 요약 캘린더에서 점 색상으로 학업 일정과 개인 일정을 구분하는 기능
- 일정 요약 캘린더에서 확인 필요 일정을 일반 일정과 다르게 표시하는 기능
- Google Calendar 외 다른 외부 캘린더 연동
- 개인 시간표 직접 연동

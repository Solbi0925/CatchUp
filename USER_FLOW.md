# CatchUp User Flow

## 1. 문서 목적

이 문서는 `IA.md`를 바탕으로 CatchUp MVP의 핵심 사용자 흐름을 정의한다.

현재 앱 구조는 아래를 기준으로 한다.

```text
온보딩
-> Google Calendar 연동
-> 메인 앱

하단 네비게이션
├─ Today
├─ Month
└─ Upload

전역 플로팅
└─ AI Mate
```

AI Mate는 하단 네비게이션의 개별 탭이 아니라, 모든 메인 화면에서 접근 가능한 플로팅 탭이다.

## 2. 첫 사용 흐름

### 목적

Google Calendar를 사용하는 사용자가 CatchUp의 핵심 기능을 이해하고, Google Calendar를 연결해 주간 계획 생성에 필요한 기본 상태를 만든다.

### 흐름

```text
앱 실행
-> Catch Up 워드마크 온보딩 확인
-> 포인트 컬러 그라디언트 인트로 노출
-> Google Calendar 연동 화면으로 자동 전환
-> Google Calendar 연동 화면
-> Google Calendar 연결
-> 연결 완료
-> 메인 앱 진입
-> Today 탭 표시
```

### 예외 흐름

```text
Google Calendar 연결 실패
-> 실패 안내 표시
-> 다시 연결 시도
-> 연결 완료 후 메인 앱 진입
```

### MVP 제한

- Google Calendar 외 다른 외부 캘린더는 연결하지 않는다.
- 개인 시간표 직접 연동은 포함하지 않는다.
- CatchUp 안에서 개인 일정을 직접 추가할 수 있다.

## 3. 학업 자료 업로드 흐름

### 목적

사용자가 강의계획서, LMS 공지, 과제 명세서 등 학업 자료를 업로드하고, AI가 추출한 정보를 직접 확인 및 수정한다.

### 흐름

```text
Upload 탭 이동
-> 학업 자료 업로드
-> 업로드한 자료 목록에 파일 표시
-> AI 추출 진행
-> 추출 결과 표시
-> 사용자가 추출 정보 확인
-> 틀린 값 또는 누락된 값 직접 수정
-> 수정 내용 저장
```

### 추출 정보

- 과제명
- 마감일
- 시험일
- 제출 방식
- 준비물
- 중요 공지
- 수업 일정 정보

### 상태 흐름

| 상태 | 의미 | 사용자 행동 |
| --- | --- | --- |
| 추출 중 | AI가 학업 자료에서 정보를 추출하는 중 | 대기 |
| 추출 완료 | 추출 결과가 표시된 상태 | 확인 후 저장 |
| 확인 필요 | 일부 값이 불확실하거나 누락된 상태 | 직접 수정 |
| 추출 실패 | 파일 인식 또는 추출 실패 | 재시도 |

### 중요 원칙

- 추출 정보 수정은 AI Mate가 대신 처리하지 않는다.
- 사용자가 화면에 표시된 필드 값을 직접 입력해 수정한다.
- 수정된 추출 정보는 플로팅 AI Mate의 주간 계획 생성 근거로 사용된다.

## 4. 주간 계획 생성 흐름

### 목적

사용자가 업로드한 학업 자료, 수정된 추출 정보, Google Calendar 개인 일정과 CatchUp에서 직접 추가한 개인 일정을 바탕으로 AI Mate를 통해 주간 계획을 생성한다.

### 흐름

```text
자료 업로드 완료
-> 플로팅 AI Mate 열기
-> 주간 계획 생성 요일과 시간 설정
-> 계획수립 요청사항 입력
-> 주간 계획 생성 요청
-> AI가 업로드 자료, 추출 정보, Google Calendar 일정 확인
-> 앞으로 4주 일정 참고
-> 이번 주 월요일~일요일 계획 생성
-> Today 탭의 이번 주 날짜별 할일에 반영
-> Month 탭의 월간 일정 요약에 반영
```

### AI가 참고하는 정보

- Upload 화면에 업로드된 학업 자료
- 사용자가 직접 수정한 AI 추출 결과
- Google Calendar 개인 일정
- CatchUp에서 직접 추가한 개인 일정
- 과제 난이도
- 예상 소요시간
- 마감일과 시험일
- 완료한 일과 미완료 항목
- 사용자의 자연어 요청사항

### 계획수립 요청사항 예시

- "일요일에는 쉬는 시간을 많이 확보해줘."
- "매일 15분 이상 영어 공부를 넣어줘."
- "팀플 회의가 많으니 과제는 짧게 나눠서 배치해줘."
- "마감이 가까운 과제부터 우선 배치해줘."

### 생성 제한 흐름

```text
사용자가 같은 주에 주간 계획 생성을 다시 요청
-> 시스템이 이번 주 생성 완료 여부 확인
-> 이미 생성 완료면 추가 생성 차단
-> 다음 생성 가능 요일과 시간 안내
-> 계획 변경은 AI Mate의 계획 조정 요청으로 유도
```

### 생성 제한 규칙

- 주간 계획 생성은 사용자가 설정한 요일과 시간에 주 1회만 가능하다.
- 한 주의 계획 범위는 월요일부터 일요일까지로 한다.
- 같은 주 안에서 주간 계획을 여러 번 생성하지 않는다.

## 5. Today 확인 흐름

### 목적

사용자가 이번 주 날짜를 선택하며 날짜별 할일을 확인하고, 기본값으로 오늘 해야 할 학습 항목과 오늘 예정 일정을 빠르게 확인한다.

### 흐름

```text
Today 탭 진입
-> 오늘 날짜가 기본 선택된 이번 주 날짜 스트립 확인
-> 필요하면 월요일~일요일 중 다른 날짜 선택
-> AI Mate 브리핑 카드 확인
-> 선택한 날짜의 할일 목록 확인
-> 선택한 날짜의 예정 일정 요약 확인
-> 추천 근거 확인
-> 완료한 항목 체크
-> 계획 변경이 필요하면 플로팅 AI Mate 열기
```

### 선택한 날짜의 할일

AI가 이번 주 계획에서 선택한 날짜로 배치한 학습 항목이다. Today 탭에 처음 들어오면 오늘 날짜가 기본 선택된다.

- 과제 진행
- 시험 공부
- 수업 준비
- 복습 또는 예습

### 선택한 날짜의 예정 일정

날짜 기준으로 선택한 날짜에 예정된 일정이다.

- 선택한 날짜가 마감인 과제
- 선택한 날짜에 제출해야 하는 항목
- 선택한 날짜에 예정된 시험
- Google Calendar 개인 일정과 CatchUp에서 직접 추가한 개인 일정

### AI Mate 브리핑

- 선택한 날짜의 할일을 시작할 수 있도록 가볍게 북돋아주는 문구를 표시한다.
- 마감이 가까운 항목, 개인 일정 충돌 가능성, 오늘 먼저 보면 좋은 항목을 짧게 요약한다.
- 정확한 완료 확률이나 성과 예측처럼 보이는 수치는 표시하지 않는다.

## 6. Month 확인 흐름

### 목적

사용자가 학업 일정과 Google Calendar 개인 일정, CatchUp에서 직접 추가한 개인 일정을 월간 캘린더로 확인한다.

### 흐름

```text
Month 탭 진입
-> 월간 일정 요약 캘린더 확인
-> 일정이 있는 날짜의 점 또는 작은 일정 칩 확인
-> 날짜 선택
-> 일정 상세 팝업 확인
-> 필요하면 개인 일정 직접 추가
-> 계획 변경이 필요하면 플로팅 AI Mate 열기
```

### 월간 캘린더 규칙

- 날짜는 일반 달력처럼 숫자로 표시한다.
- 일정이 있는 날짜는 숫자 아래 점 또는 작은 일정 칩으로 표시한다.
- 일정 표시는 다양한 색상을 쓰지 않고 CatchUp 포인트 컬러의 명도 차이로만 구분한다.
- 학업 일정과 개인 일정을 서로 다른 카테고리 색으로 나누는 기능은 초기 구현에서 제외하고, 출처 텍스트로 구분한다.
- 날짜를 선택하면 해당 날짜의 일정 상세 팝업을 표시한다.
- 사용자는 CatchUp 안에서 개인 일정의 제목, 날짜, 시간을 직접 입력해 추가할 수 있다. 이 기능은 주간 계획 항목 직접 수정 기능이 아니다.
- 월간 화면은 일정 확인용이며, 월간 세부 자동 계획 생성 기능은 제공하지 않는다.

## 7. 계획 조정 흐름

### 목적

사용자가 생성된 계획 중 수정하고 싶은 부분을 플로팅 AI Mate를 통해 자연어로 조정한다.

### 흐름

```text
Today 또는 Month 확인
-> 수정하고 싶은 부분 발견
-> 플로팅 AI Mate 열기
-> 계획 조정 요청 입력
-> 오늘의 조정 잔여 횟수 확인
-> 잔여 횟수가 있으면 AI가 계획 조정
-> 조정 결과와 추천 근거 표시
-> Today의 날짜별 할일 / Month 일정 요약에 반영
```

### 조정 요청 예시

- "오늘 할 일을 조금 줄여줘."
- "수요일에 개인 일정이 많으니 그날 할 일을 줄여줘."
- "시험 공부 시간을 늘려줘."
- "알고리즘 과제를 더 앞쪽에 배치해줘."

### 조정 제한 흐름

```text
사용자가 하루 10회를 초과해 계획 조정 요청
-> 오늘의 잔여 횟수 0회 확인
-> 추가 조정 요청 차단
-> 다음 날 다시 가능하다는 안내 표시
```

### 조정 제한 규칙

- AI Mate를 통한 일일/주간 계획 조정 요청은 하루에 최대 10번만 가능하다.
- 사용자가 계획 항목의 시간, 날짜, 우선순위, 내용을 직접 손으로 수정하는 기능은 MVP에서 제외한다. 계획의 생성과 수정은 AI Mate로만 처리한다.

## 8. 추천 근거 확인 흐름

### 목적

사용자가 AI가 왜 특정 학습 항목을 추천했는지 이해하도록 한다.

### 흐름

```text
Today의 날짜별 할일 또는 Month의 일정 상세에서 항목 확인
-> 추천 근거 요약 확인
-> 더 자세한 이유가 필요하면 플로팅 AI Mate 열기
-> 추천 이유 질문
-> AI가 근거 설명
```

### 추천 근거 요소

- 마감일까지 남은 기간
- 예상 소요시간
- 중요도
- Google Calendar 개인 일정과의 충돌 여부
- 시험과의 연관성

## 9. MVP 범위 확인

- 하단 네비게이션은 `Today`, `Month`, `Upload` 3개 탭으로 구성한다.
- AI Mate는 개별 탭이 아니라 전역 플로팅 탭으로 제공한다.
- Upload 화면의 업로드 자료와 수정된 추출 정보는 AI Mate의 주간 계획 생성 근거로 사용된다.
- 주간 계획 생성은 주 1회만 가능하다.
- AI Mate 계획 조정 요청은 하루에 최대 10번만 가능하다.
- Google Calendar 외 다른 외부 캘린더는 연결하지 않는다.
- 전체 월간 세부 자동 계획 생성은 제공하지 않는다.

# Catchup IA

## 목적

Catchup MVP의 화면 구조를 3개 이하의 페이지로 정리한다. 모든 페이지는 `PRD.md`의 1차 MVP 범위 안에서만 설계한다.

## 메뉴 구조

- 자료 등록
- 추출 정보 확인
- 학습 계획

## 페이지 구조

### 1. 자료 등록

**목적**  
사용자가 학업 자료와 개인 일정 정보를 등록해 AI가 계획을 만들 수 있는 입력값을 준비한다.

**포함 정보**

- 강의계획서 파일 업로드
- LMS 공지 파일 업로드
- 과제 명세서 업로드
- 시간표 파일 업로드
- 최초 구현 파일 형식은 PDF 또는 이미지 중 한 가지
- Google Calendar 연결 또는 개인 시간표 입력
- 업로드 상태
- AI 추출 시작 버튼

### 2. 추출 정보 확인

**목적**  
AI가 추출한 학업 정보를 사용자가 확인하고 잘못된 내용을 수정한다.

**포함 정보**

- 과제명
- 마감일
- 시험일
- 제출 방식
- 준비물
- 중요 공지
- 수업 일정 또는 시간표 정보
- “확인 필요” 표시
- 직접 수정 기능
- 학습 계획 생성 버튼

### 3. 학습 계획

**목적**  
사용자가 오늘/이번 주 학습 계획을 확인하고, 완료 체크와 AI 챗봇으로 계획을 조정한다.

**포함 정보**

- 오늘 할 일
- 이번 주 할 일
- 앞으로 4주간의 주요 마감일, 시험일, 제출일
- 마감 임박 일정
- 과제 난이도
- 예상 소요시간
- 개인 일정과의 충돌 여부
- 완료 체크
- 일정 시간, 날짜, 우선순위, 할 일 내용 수정
- 수정 또는 완료 체크 후 남은 일정 기본 재정렬
- 주간 계획 생성
- 횟수 제한이 있는 AI 챗봇 계획 수정 요청
- AI 추천 근거

## MVP 범위 점검

- 페이지 수는 3개로 제한했다.
- `PRD.md`의 핵심 기능인 학업 정보 추출, 개인 일정 기반 계획 생성, AI 챗봇 기반 조정을 모두 포함했다.
- 모든 LMS 자동 연동, 완벽한 OCR, 장기 학습 성과 예측, 정확한 완료 확률 계산 페이지는 만들지 않았다.
- 친구 공동 일정, 학교 행정 시스템 연동, 모바일 앱 전용 화면은 포함하지 않았다.
- 월간 계획 상세 페이지는 만들지 않고, 앞으로 4주간의 마감일과 시험일은 학습 계획 페이지에서 참고 정보로만 다룬다.
- 매일 AI가 전체 계획을 다시 생성하는 페이지는 만들지 않고, 완료 체크 후 기본 재정렬만 포함했다.
- 따라서 현재 IA는 `PRD.md`의 10주 MVP 범위와 맞다.

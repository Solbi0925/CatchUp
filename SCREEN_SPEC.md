# Catchup Screen Spec

## 목적

`IA.md`의 3개 페이지를 기준으로 각 화면의 주요 요소와 상태를 정리한다. 아직 코드는 만들지 않는다.

## 공통 규칙

- 1차 MVP는 웹 화면 기준으로 작성한다.
- 실제 학생 개인정보는 사용하지 않는다.
- AI 추천은 참고용 계획으로 표현한다.
- 정확한 완료 확률 대신 `우선순위 높음`, `완료 위험도 높음` 같은 정성 표현을 사용한다.

## 1. 자료 등록 화면

### 목적

사용자가 학업 자료와 개인 일정 정보를 등록해 AI 추출과 계획 생성을 시작할 수 있게 한다.

### 주요 요소

- 화면 제목: 자료 등록
- 파일 업로드 영역
- 지원 파일 형식 안내: 최초 MVP는 PDF 또는 이미지 중 한 가지
- 업로드 파일 목록
- Google Calendar 연결 버튼
- 개인 시간표 직접 입력 영역
- AI 추출 시작 버튼
- 다음 단계 안내

### Loading

- 파일 업로드 중: “파일을 업로드하는 중입니다.”
- Google Calendar 연결 중: “캘린더를 연결하는 중입니다.”
- AI 추출 시작 후: “학업 정보를 읽는 중입니다.”

### Empty

- 업로드 전: “강의계획서, LMS 공지, 과제 명세서, 시간표 파일을 등록해주세요.”
- 캘린더 미연결: “Google Calendar를 연결하거나 개인 시간표를 직접 입력해주세요.”

### Error

- 파일 업로드 실패: “파일을 업로드하지 못했습니다. 파일 형식과 크기를 확인해주세요.”
- 지원하지 않는 파일 형식: “현재 MVP에서는 지원하지 않는 파일 형식입니다.”
- 캘린더 연결 실패: “Google Calendar를 연결하지 못했습니다. 개인 시간표 입력으로 진행할 수 있습니다.”

### Success

- 파일 업로드 완료: “파일이 등록되었습니다.”
- 캘린더 연결 완료: “Google Calendar가 연결되었습니다.”
- AI 추출 시작 가능: `AI 추출 시작` 버튼 활성화

## 2. 추출 정보 확인 화면

### 목적

AI가 추출한 학업 정보를 사용자가 확인하고 수정한 뒤 학습 계획 생성을 요청한다.

### 주요 요소

- 화면 제목: 추출 정보 확인
- 추출 결과 목록
- 과제명 입력/수정 필드
- 마감일 입력/수정 필드
- 시험일 입력/수정 필드
- 제출 방식 입력/수정 필드
- 준비물 입력/수정 필드
- 중요 공지 입력/수정 필드
- 수업 일정 또는 시간표 정보
- `확인 필요` 배지
- 항목 추가 버튼
- 학습 계획 생성 버튼

### Loading

- 추출 결과 불러오는 중: “추출한 정보를 불러오는 중입니다.”
- 계획 생성 요청 중: “오늘과 이번 주 계획을 만드는 중입니다.”

### Empty

- 추출 결과 없음: “아직 추출된 학업 정보가 없습니다.”
- 특정 항목 없음: “마감일이 없는 항목입니다. 필요하면 직접 입력해주세요.”

### Error

- 추출 실패: “일부 정보를 추출하지 못했습니다. 확인 필요 항목을 직접 수정해주세요.”
- 저장 실패: “수정한 정보를 저장하지 못했습니다. 다시 시도해주세요.”
- 계획 생성 불가: “계획 생성을 위해 마감일 또는 시험일을 1개 이상 입력해주세요.”

### Success

- 추출 완료: “학업 정보가 추출되었습니다.”
- 수정 저장 완료: “수정한 정보가 저장되었습니다.”
- 계획 생성 요청 완료: 학습 계획 화면으로 이동

## 3. 학습 계획 화면

### 목적

사용자가 오늘/이번 주 계획을 확인하고, 완료 체크와 직접 수정 또는 AI 챗봇으로 계획을 조정한다.

### 주요 요소

- 화면 제목: 학습 계획
- 오늘 할 일 목록
- 이번 주 할 일 목록
- 앞으로 4주간의 주요 마감일, 시험일, 제출일
- 마감 임박 표시
- 과제 난이도
- 예상 소요시간
- 개인 일정 충돌 표시
- 완료 체크박스
- 일정 시간, 날짜, 우선순위, 할 일 내용 수정 기능
- 남은 일정 기본 재정렬 결과
- 주간 계획 생성 버튼
- AI 챗봇 입력창
- 챗봇 수정 요청 남은 횟수
- AI 추천 근거 영역

### Loading

- 계획 생성 중: “앞으로 4주간의 일정을 참고해 이번 주 계획을 만드는 중입니다.”
- 완료 체크 반영 중: “남은 일정을 정리하는 중입니다.”
- 챗봇 수정 요청 중: “요청사항을 반영해 계획을 조정하는 중입니다.”

### Empty

- 오늘 할 일 없음: “오늘 예정된 학업 할 일이 없습니다.”
- 이번 주 할 일 없음: “이번 주에 등록된 학업 일정이 없습니다.”
- 4주 일정 없음: “앞으로 4주간 등록된 마감일이나 시험일이 없습니다.”
- 추천 근거 없음: “계획을 생성하면 추천 근거가 표시됩니다.”

### Error

- 계획 생성 실패: “계획을 생성하지 못했습니다. 일정 정보를 확인해주세요.”
- 완료 체크 반영 실패: “완료 상태를 반영하지 못했습니다. 다시 시도해주세요.”
- 직접 수정 저장 실패: “수정한 일정을 저장하지 못했습니다.”
- 챗봇 요청 횟수 초과: “오늘 사용할 수 있는 AI 수정 요청 횟수를 모두 사용했습니다.”
- 챗봇 응답 실패: “AI가 요청을 처리하지 못했습니다. 직접 수정하거나 다시 시도해주세요.”

### Success

- 계획 생성 완료: “이번 주 학습 계획이 생성되었습니다.”
- 완료 체크 반영 완료: “남은 일정이 정리되었습니다.”
- 직접 수정 저장 완료: “수정한 일정이 반영되었습니다.”
- 챗봇 수정 완료: “요청사항을 반영해 계획을 수정했습니다.”

## MVP 범위 점검

- 화면은 `IA.md`의 3개 페이지를 넘지 않는다.
- 파일 업로드, 추출 정보 확인, 오늘/이번 주 계획, 챗봇 수정 요청만 포함했다.
- 앞으로 4주간의 일정은 계획 생성 참고 정보로만 사용한다.
- 매일 AI가 전체 계획을 자동 재생성하는 기능은 포함하지 않았다.
- 모든 LMS 자동 연동, 친구 공동 일정, 모바일 앱 정식 출시 화면은 포함하지 않았다.

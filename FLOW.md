# Catchup User Flow

## 목적

`IA.md`의 3개 페이지를 기준으로 사용자의 기본 흐름과 실패 상태를 정리한다. MVP 범위는 `PRD.md`의 1차 구현 범위를 넘지 않는다.

## 기본 사용자 흐름

### 1. 자료 등록

1. 사용자가 `자료 등록` 페이지에 들어온다.
2. 강의계획서, LMS 공지, 과제 명세서, 시간표 중 필요한 파일을 업로드한다.
3. 최초 MVP에서는 PDF 또는 이미지 중 한 가지 형식만 지원한다.
4. 사용자가 Google Calendar를 연결하거나 개인 시간표를 직접 입력한다.
5. 사용자가 `AI 추출 시작` 버튼을 누른다.
6. 시스템은 업로드된 파일에서 텍스트를 읽고 학업 정보를 추출한다.

### 2. 추출 정보 확인

1. 사용자가 `추출 정보 확인` 페이지에서 AI가 추출한 정보를 확인한다.
2. 과제명, 마감일, 시험일, 제출 방식, 준비물, 중요 공지, 수업 일정 정보를 확인한다.
3. 잘못 추출된 항목은 사용자가 직접 수정한다.
4. 불확실한 항목은 `확인 필요` 상태로 표시된다.
5. 사용자가 `학습 계획 생성` 버튼을 누른다.

### 3. 학습 계획

1. 시스템은 수업 시간표, 개인 일정, 학업 일정, 과제 난이도, 예상 소요시간을 함께 고려한다.
2. 시스템은 오늘 기준 앞으로 4주간의 마감일, 시험일, 제출일을 참고한다.
3. 시스템은 오늘 할 일과 이번 주 할 일을 생성한다.
4. 사용자는 추천 근거를 확인한다.
5. 사용자는 일정의 시간, 날짜, 우선순위, 할 일 내용을 직접 수정할 수 있다.
6. 사용자가 완료 체크를 하면 남은 일정은 기본 규칙에 따라 재정렬된다.
7. 사용자는 제한된 횟수 안에서 AI 챗봇에게 계획 수정을 요청할 수 있다.

## 실패 상태

### 파일 업로드 실패

- 원인: 지원하지 않는 파일 형식, 파일 용량 초과, 네트워크 오류
- 사용자 안내: “파일을 업로드하지 못했습니다. 지원 형식과 파일 크기를 확인해주세요.”
- 다음 행동: 다시 업로드하거나 다른 파일을 선택한다.

### 파일 인식 실패

- 원인: 이미지 품질 낮음, 텍스트가 흐림, PDF 텍스트 추출 실패
- 사용자 안내: “일부 내용을 읽지 못했습니다. 확인 필요 항목을 직접 입력해주세요.”
- 다음 행동: 사용자가 누락된 과제명, 마감일, 시험일 등을 직접 입력한다.

### 학업 정보 추출 실패

- 원인: 문서 안에 마감일이나 과제 정보가 명확하지 않음
- 사용자 안내: “추출된 정보가 부족합니다. 필요한 항목을 직접 추가해주세요.”
- 다음 행동: 사용자가 수동으로 학업 일정을 추가한 뒤 계획 생성을 진행한다.

### Google Calendar 연결 실패

- 원인: 권한 거부, 연결 오류, 계정 인증 실패
- 사용자 안내: “Google Calendar를 연결하지 못했습니다. 개인 시간표를 직접 입력할 수 있습니다.”
- 다음 행동: 다시 연결하거나 개인 시간표 입력으로 대체한다.

### 학습 계획 생성 실패

- 원인: 학업 일정 또는 개인 일정 정보 부족, AI 응답 실패
- 사용자 안내: “계획을 생성하지 못했습니다. 일정 정보를 확인한 뒤 다시 시도해주세요.”
- 다음 행동: 추출 정보와 시간표를 수정한 뒤 다시 생성한다.

### 챗봇 수정 요청 실패

- 원인: 요청 횟수 초과, AI 응답 실패, 요청 내용이 모호함
- 사용자 안내: “지금은 계획을 수정하지 못했습니다. 요청 내용을 더 구체적으로 작성하거나 나중에 다시 시도해주세요.”
- 다음 행동: 직접 일정을 수정하거나 남은 요청 횟수를 확인한다.

### 빈 계획 상태

- 원인: 앞으로 4주 안에 마감일, 시험일, 제출일이 없음
- 사용자 안내: “앞으로 4주간 등록된 학업 일정이 없습니다.”
- 다음 행동: 새 자료를 업로드하거나 일정을 직접 추가한다.

## MVP 범위 밖 흐름

- 모든 LMS와 학교 시스템 자동 연동
- 모든 파일 형식 완벽 지원
- 매일 AI가 전체 계획을 자동 재생성하는 흐름
- 친구와 함께 일정을 관리하는 흐름
- 모바일 앱 전용 흐름

# CatchUp Screen Spec

## 1. 문서 목적

이 문서는 `IA.md`와 `USER_FLOW.md`를 바탕으로 CatchUp MVP의 화면별 구성 요소, 표시 정보, 사용자 행동, 상태를 정의한다.

현재 화면 구조는 아래를 기준으로 한다.

```text
온보딩
-> Google Calendar 연동
-> 메인 앱

하단 네비게이션
├─ Today
├─ Month
└─ Upload

전역 플로팅
└─ AI Mate
```

## 2. 공통 화면 규칙

### 네비게이션

- 메인 앱은 하단 네비게이션 바를 사용한다.
- 하단 네비게이션에는 `Today`, `Month`, `Upload` 3개 탭만 둔다.
- AI Mate는 하단 네비게이션의 개별 탭으로 만들지 않는다.
- AI Mate는 모든 메인 화면에서 접근 가능한 전역 플로팅 버튼으로 제공한다.
- 이전의 가로 스크롤 페이지 이동은 사용하지 않는다.

### 공통 데이터 표시 원칙

- 학업 일정과 개인 일정은 출처를 구분해 표시한다.
- AI가 추출한 정보 중 불확실한 값은 `확인 필요`로 표시한다.
- 추천 근거는 짧게 먼저 보여주고, 필요 시 플로팅 AI Mate에서 자세히 질문할 수 있다.
- 수치형 완료 확률은 표시하지 않는다.
- Today 할일과 주간 계획 항목의 직접 수정 UI는 MVP에서 제공하지 않는다. 계획의 생성과 수정은 AI Mate로만 처리한다.
- 주요 포인트 컬러는 보라 계열로 통일한다.
- 월간 일정 표시는 다양한 카테고리 색상을 쓰지 않고, 같은 포인트 컬러의 명도 단계로만 시각적 구분을 만든다.
- 학업 일정과 개인 일정의 의미 구분은 색상보다 텍스트 유형과 출처로 표시한다.

### 공통 제한

- 주간 계획 생성은 사용자가 설정한 요일과 시간에 주 1회만 가능하다.
- 한 주의 계획 범위는 월요일부터 일요일까지다.
- AI Mate를 통한 일일/주간 계획 조정 요청은 하루에 최대 10번만 가능하다.
- Google Calendar 외 다른 외부 캘린더는 연결하지 않는다.

### 공통 상태

| 상태 | 설명 |
| --- | --- |
| 빈 상태 | 아직 업로드, 연결, 계획 생성이 완료되지 않은 상태 |
| 로딩 상태 | 자료 추출, 일정 불러오기, 계획 생성 중인 상태 |
| 확인 필요 | AI 추출 결과 중 사용자가 검토해야 하는 항목이 있는 상태 |
| 오류 상태 | 업로드 실패, 추출 실패, 캘린더 연결 실패 등 |
| 정상 상태 | 일정과 계획이 표시되는 상태 |
| 제한 도달 | 주간 계획 생성 또는 AI Mate 조정 요청 가능 횟수를 모두 사용한 상태 |

## 3. 온보딩 화면

### 목적

사용자가 CatchUp의 첫인상과 서비스 무드를 빠르게 인지한 뒤 Google Calendar 연동으로 자연스럽게 이동하도록 한다.

### 화면 구성

```text
[포인트 컬러 그라디언트 배경]
[Catch Up 워드마크]
[짧은 한 줄 설명]
[자동 전환 상태]
```

### 표시 정보

- 워드마크: Catch Up
- 한 줄 설명: 오늘과 이번 주 학업 계획을 한눈에
- 보라 계열 포인트 컬러 그라디언트
- 은은한 빛 번짐 또는 별빛 보조 그래픽
- 다음 화면으로 이동 중임을 느낄 수 있는 짧은 로딩 또는 전환 상태

### 사용자 행동

- 앱 소개 확인
- 별도 조작 없이 Google Calendar 연동 화면으로 자동 이동

### 전환 규칙

- 첫 실행 시 온보딩 화면을 짧게 노출한 뒤 Google Calendar 연동 화면으로 자동 전환한다.
- 프로젝트명과 문서명은 `CatchUp`을 유지하되, 온보딩 워드마크에서는 `Catch Up` 표기를 사용할 수 있다.

## 4. Google Calendar 연동 화면

### 목적

사용자의 개인 일정을 불러와 학업 일정과 함께 주간 계획 생성에 반영할 수 있도록 한다.

### 화면 구성

```text
[Google Calendar 연동 안내]
[Google Calendar 아이콘과 연결 그래픽]
[연동 필요 이유]
[Google Calendar 연결 버튼]
[연결 상태]
```

### 표시 정보

- Google Calendar 연동 안내
- 개인 일정과 학업 일정 충돌 방지 설명
- 주간 계획 생성 시 개인 일정이 반영된다는 설명
- 연결 시 반영되는 정보
  - 개인 일정과 수업 시간을 주간 계획에 반영
  - 충돌 없는 일정으로 하루 만들기
  - 언제든 연결 해제 가능
- 연결 상태
  - 연결 전
  - 연결 중
  - 연결 완료
  - 연결 실패

### 사용자 행동

- Google Calendar 연결 버튼 선택
- 권한 승인
- 연결 실패 시 재시도
- 연결 완료 후 메인 앱 진입

### 오류 상태

- 연결 실패 시 재시도 안내를 표시한다.
- Google Calendar 연결이 완료되지 않으면 주간 계획 생성 전에 연결을 완료하도록 안내한다.

## 5. 메인 공통 레이아웃

### 화면 구성

```text
[현재 탭 콘텐츠]

[플로팅 AI Mate 버튼]

[하단 네비게이션]
- Today
- Month
- Upload
```

### 플로팅 AI Mate 버튼

- 모든 메인 탭에서 표시한다.
- 하단 네비게이션 오른쪽 위 또는 화면 오른쪽 하단에 배치한다.
- 사용자가 누르면 AI Mate 패널을 연다.
- 버튼은 주요 콘텐츠를 가리지 않도록 배치한다.

## 6. 탭 1: Today

### 목적

AI가 이번 주 계획에서 선택한 날짜로 배치한 학습 항목과 예정 일정을 빠르게 확인한다. 첫 진입 시 오늘 날짜가 기본 선택된다.

### 화면 구성

```text
[상단 인사와 오늘 날짜]

[이번 주 날짜 선택 영역]

[AI Mate 브리핑 카드]

[선택한 날짜의 할일]

[선택한 날짜의 예정 일정 요약]

[플로팅 AI Mate 버튼]
```

### 섹션 1: 상단 인사와 날짜

#### 표시 정보

- 짧은 인사 문구
- 날짜
- 요일
- 선택한 날짜의 할일 개수
- 선택한 날짜의 예정 일정 개수

### 섹션 2: 이번 주 날짜 선택 영역

#### 정의

이번 주 월요일부터 일요일까지를 한 줄로 배치해 사용자가 날짜별 할일을 확인할 수 있게 한다.

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 요일 | 월, 화, 수, 목, 금, 토, 일 |
| 날짜 | 해당 주의 날짜 숫자 |
| 선택 상태 | 선택한 날짜를 포인트 컬러 또는 진한 배경으로 강조 |
| 오늘 상태 | 오늘 날짜를 기본 선택하고 필요 시 별도 표시 |
| 일정 존재 여부 | 작은 점 또는 약한 포인트 컬러로 일정 존재를 표시 |

#### 사용자 행동

- 날짜 선택
- 선택한 날짜의 할일 목록과 예정 일정 확인

### 섹션 3: AI Mate 브리핑 카드

#### 역할

선택한 날짜의 핵심 행동을 가볍게 북돋아주고, 오늘 집중하면 좋은 항목을 짧게 알려준다.

#### 표시 정보

- AI Mate 라벨
- 격려 문구
- 선택한 날짜의 핵심 요약
- 마감이 가까운 항목 또는 먼저 시작하면 좋은 항목
- 추천 근거 한 줄

#### 표현 원칙

- 행동을 부드럽게 독려하는 톤을 사용한다.
- 정확한 완료 확률, 성과 예측, 점수 예측은 표시하지 않는다.
- 긴 설명은 넣지 않고 필요 시 AI Mate 패널에서 자세히 묻도록 한다.

### 섹션 4: 선택한 날짜의 할일

#### 정의

선택한 날짜의 할일은 AI가 앞으로 4주간의 일정, 난이도, 마감일, 우선순위, 예상 소요시간, Google Calendar 개인 일정과 CatchUp에서 직접 추가한 개인 일정을 고려해 주간 계획에서 해당 날짜로 배치한 학습 항목이다.

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 과목명 또는 자료 출처 | 어떤 수업 또는 자료에서 나온 일인지 표시 |
| 할일 제목 | 오늘 해야 할 학습 행동 |
| 예상 소요시간 | 계획 이해를 돕는 시간 정보 |
| 우선순위 | 높음, 보통, 낮음 |
| 완료 여부 | 완료 체크 상태 |
| 추천 근거 요약 | 왜 오늘 해야 하는지 한 문장으로 표시 |

#### 할일 유형

- 과제 진행
- 시험 공부
- 수업 준비
- 복습 또는 예습

#### 사용자 행동

- 선택한 날짜의 할일 확인
- 완료 체크
- 추천 근거 확인
- 계획 변경이 필요하면 플로팅 AI Mate 열기

#### 빈 상태

- 주간 계획이 없으면 `아직 이번 주 할일이 없습니다.`를 표시한다.
- 플로팅 AI Mate에서 주간 계획 생성을 요청하도록 안내한다.

### 섹션 5: 선택한 날짜의 예정 일정 요약

#### 정의

선택한 날짜의 예정 일정은 날짜 기준으로 해당 날짜에 예정된 일정이다. AI가 임의로 배치한 학습 항목이 아니라, 학업 자료에서 추출된 일정, Google Calendar 개인 일정 또는 CatchUp에서 직접 추가한 개인 일정이다.

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 일정명 | 마감, 제출, 시험, 개인 일정 제목 |
| 시간 | 오늘 중 예정 시간 또는 종일 |
| 유형 | 마감, 제출, 시험, 중요 공지, 개인 일정 |
| 출처 | 업로드 자료, Google Calendar 또는 CatchUp 직접 입력 |

#### 사용자 행동

- 선택한 날짜의 예정 일정 확인
- 일정 출처 확인

### 탭 1에서 제외하는 기능

- 주간 계획 생성 입력
- 계획 조정 입력
- 학업 자료 업로드
- 할일의 시간, 날짜, 우선순위, 내용 직접 수정

## 7. 탭 2: Month

### 목적

학업 일정과 Google Calendar 개인 일정, CatchUp에서 직접 추가한 개인 일정을 월간 캘린더로 확인한다. 이번 주 요일별 할일 확인은 Today 탭의 날짜 선택 영역에서 처리한다.

### 화면 구성

```text
[월간 일정 요약 캘린더]

[선택 날짜 일정 상세 요약]

[플로팅 AI Mate 버튼]
```

### 섹션 1: 월간 일정 요약 캘린더

#### 역할

월간 단위로 학업 일정과 개인 일정을 확인한다. 월간 화면은 일정 확인용이며, 월간 세부 자동 계획을 생성하지 않는다.

#### 표시 정보

- AI가 학업 자료에서 추출한 마감일
- 제출일
- 시험일
- 중요 공지 일정
- Google Calendar 개인 일정과 CatchUp에서 직접 추가한 개인 일정

#### 표시 규칙

- 날짜는 일반 달력처럼 숫자로 표시한다.
- 일정이 있는 날짜는 날짜 숫자 아래 점 또는 작은 일정 칩으로 표시한다.
- 일정 1개는 점 1개, 일정 2개는 점 2개, 일정 3개는 점 3개로 표시한다.
- 점은 일정의 상세 내용을 직접 보여주지 않고, 해당 날짜에 일정이 있다는 신호로만 사용한다.
- 일정 칩이나 점은 CatchUp 포인트 컬러의 명도 단계로만 구분한다.
- 초기 구현에서는 색상으로 학업 일정과 개인 일정을 구분하지 않는다.
- 초기 구현에서는 확인 필요 일정을 일반 일정과 다르게 표시하지 않는다.

#### 사용자 행동

- 날짜 선택
- 선택한 날짜의 일정 상세 팝업 확인
- 일정 출처 확인
- 개인 일정 직접 추가

### 섹션 2: 일정 상세 팝업

날짜를 선택하면 해당 날짜의 일정 상세 요약을 팝업으로 표시한다.

```text
7일
- 알고리즘 과제 제출 마감일
- Google Calendar 개인 일정
```

#### 팝업 표시 정보

| 항목 | 설명 |
| --- | --- |
| 선택 날짜 | 사용자가 선택한 날짜 |
| 일정명 | 해당 날짜의 일정 제목 |
| 유형 | 마감, 제출, 시험, 중요 공지, 개인 일정 |
| 출처 | 업로드 자료 또는 Google Calendar |

### 탭 2에서 제외하는 기능

- 주간 계획 생성 입력
- 계획 조정 입력
- 학업 자료 업로드
- 주간 계획 항목의 시간, 날짜, 우선순위, 내용 직접 수정
- 전체 월간 세부 계획 생성

### 개인 일정 직접 추가

- Month 탭에서 개인 일정의 제목, 날짜, 시작·종료 시간을 직접 입력해 추가할 수 있다.
- 직접 추가한 개인 일정은 Google Calendar 일정과 함께 Today, Month, AI Mate의 계획 생성 근거에 반영한다.
- 이 기능은 AI가 생성한 주간 계획 항목을 직접 수정하는 기능이 아니다.

## 8. 탭 3: Upload

### 목적

학업 자료를 업로드하고 AI가 추출한 정보를 직접 확인 및 수정한다. 업로드 자료와 수정된 추출 정보는 플로팅 AI Mate가 주간 계획을 생성할 때 사용하는 핵심 근거가 된다.

### 화면 구성

```text
[학업 자료 업로드]

[업로드한 자료 목록]

[AI 추출 결과 확인 및 수정]

[플로팅 AI Mate 버튼]
```

### 섹션 1: 학업 자료 업로드

#### 표시 정보

- 업로드 영역
- 점선 테두리의 드롭 영역
- 업로드 아이콘
- 지원 파일 형식 안내
- 업로드 진행 상태

#### 지원 범위

- MVP에서는 PDF와 이미지 파일 형식을 모두 지원한다.
- PDF와 이미지 외의 파일 형식은 지원하지 않는다.

#### 사용자 행동

- 강의계획서 업로드
- LMS 공지 업로드
- 과제 명세서 업로드
- 업로드 실패 시 재시도

### 섹션 2: 업로드한 자료 목록

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 파일명 | 업로드한 학업 자료 이름 |
| 파일 형식 | PDF 또는 이미지 |
| 파일 크기 | 업로드 자료 크기 |
| 업로드 날짜 | 파일을 올린 날짜 |
| 추출 상태 | 추출 중, 추출 완료, 확인 필요, 추출 실패 |
| 확인 필요 여부 | 사용자가 검토해야 하는지 표시 |

#### 사용자 행동

- 업로드한 파일 확인
- 추출 상태 확인
- 확인 필요 파일 선택

### 섹션 3: AI 추출 결과 확인 및 수정

#### 역할

AI가 학업 자료에서 추출한 정보를 사용자에게 보여주고, 사용자가 틀린 부분을 직접 입력해 수정한다.

#### 표시 정보

| 필드 | 설명 |
| --- | --- |
| 과제명 | 추출된 과제 이름 |
| 마감일 | 과제 또는 제출 마감일 |
| 시험일 | 시험 예정일 |
| 제출 방식 | 온라인 제출, LMS 제출 등 |
| 준비물 | 수업 또는 과제 준비물 |
| 중요 공지 | 놓치면 안 되는 공지 |
| 수업 일정 정보 | 수업 관련 일정 |

#### 상태

| 상태 | 설명 |
| --- | --- |
| 추출 중 | AI가 정보를 추출하는 중 |
| 추출 완료 | 사용자가 확인할 수 있는 상태 |
| 확인 필요 | 일부 값이 불확실하거나 누락된 상태 |
| 저장 완료 | 사용자가 수정한 정보가 반영된 상태 |

#### 사용자 행동

- 추출 결과 확인
- 틀린 값 직접 수정
- 누락된 값 직접 입력
- 확인 필요 항목 검토
- 수정 내용 저장
- 플로팅 AI Mate를 열어 업로드 자료 기반 주간 계획 생성 요청

#### 제외 사항

- AI Mate가 추출 정보를 대신 수정하는 흐름은 포함하지 않는다.
- 추출 결과 수정은 사용자의 직접 입력으로 처리한다.

## 9. 전역 플로팅 AI Mate

### 목적

사용자가 어떤 메인 화면에 있든 AI Mate를 열어 주간 계획 생성, 계획수립 요청사항 입력, 생성된 계획 조정, 추천 근거 질문을 할 수 있도록 한다.

### 화면 구성

```text
[플로팅 AI Mate 버튼]

버튼 선택 시:
[AI Mate 패널]
- 패널 헤더와 닫기 버튼
- 대화 영역
- 자연어 입력 영역
- 주간 계획 생성 설정 요약
- 주간 계획 생성 상태
- 계획 조정 잔여 횟수
```

### 플로팅 버튼

#### 표시 규칙

- 모든 메인 탭에서 표시한다.
- 하단 네비게이션 오른쪽 위 또는 화면 오른쪽 하단에 배치한다.
- 현재 콘텐츠와 하단 네비게이션을 가리지 않도록 배치한다.

#### 사용자 행동

- 버튼 선택
- AI Mate 패널 열기
- 패널 닫기

### AI Mate 패널

#### 표시 정보

| 항목 | 설명 |
| --- | --- |
| 패널 헤더 | `AI Mate` 이름, 조정 잔여 횟수, 닫기 버튼 표시 |
| 대화 영역 | AI와 사용자의 메시지 표시 |
| 자연어 입력 영역 | 계획 생성 요청, 계획 조정 요청, 질문 입력 |
| 주간 계획 생성 요일 | 사용자가 설정한 생성 요일 |
| 주간 계획 생성 시간 | 사용자가 설정한 생성 시간 |
| 다음 생성 예정 시각 | 다음 주간 계획 생성 가능 시각 |
| 이번 주 생성 여부 | 이번 주 계획 생성 완료 여부 |
| 조정 잔여 횟수 | 오늘의 AI Mate 계획 조정 가능 횟수 |

#### AI가 참고하는 정보

- 자료 업로드 화면에 업로드된 학업 자료
- 사용자가 직접 수정한 AI 추출 결과
- Google Calendar 개인 일정
- 완료한 일과 미완료 항목
- 사용자의 자연어 요청사항

#### 입력 가능 요청

- 주간 계획 생성 요청
- 계획수립 요청사항 입력
- 오늘 계획 조정 요청
- 이번 주 계획 조정 요청
- 추천 근거 질문

#### 요청 예시

- "이번 주 계획 짜줘."
- "자료 업로드 화면에 올린 강의계획서랑 과제 명세서 기반으로 이번 주 계획 짜줘."
- "일요일에는 쉬는 시간을 많이 확보해줘."
- "오늘 할 일을 조금 줄여줘."
- "수요일에 개인 일정이 많으니 그날 할 일을 줄여줘."
- "왜 이 과제를 오늘 해야 해?"

#### 응답에 포함할 정보

- 생성 또는 조정된 계획 요약
- 반영한 사용자 요청사항
- 추천 근거
- 충돌 또는 위험도 안내
- 오늘 남은 계획 조정 횟수

### 제한 상태

| 상태 | 화면 동작 |
| --- | --- |
| 주간 계획 생성 전 | 설정된 요일과 시간이 되면 주간 계획 생성 가능 |
| 이번 주 생성 완료 | 같은 주 안의 추가 주간 계획 생성 요청 차단 |
| 조정 잔여 1~10회 | AI Mate 계획 조정 요청 가능 |
| 조정 잔여 0회 | 추가 조정 요청 차단, 다음 날 재사용 가능 안내 |

### 표현 제한

- 정확한 완료 확률 수치는 표시하지 않는다.
- `우선순위 높음`, `완료 위험도 높음`처럼 정성적 표현을 사용한다.

## 10. 주요 데이터 표시 구분

### 선택한 날짜의 할일과 예정 일정 구분

| 구분 | 선택한 날짜의 할일 | 선택한 날짜의 예정 일정 |
| --- | --- | --- |
| 기준 | AI가 선택한 날짜로 배치한 학습 행동 | 날짜 기준으로 선택한 날짜에 예정된 일정 |
| 생성 방식 | 주간 계획 생성 결과 | 학업 자료 추출, Google Calendar 또는 CatchUp 직접 입력 |
| 예시 | 과제 2시간 진행, 시험 공부 40분 | 과제 마감, 시험, 개인 약속 |
| 사용자 행동 | 완료 체크, AI Mate에 계획 조정 요청 | 일정 확인, 출처 확인 |

### 학업 일정과 개인 일정 구분

| 구분 | 학업 일정 | 개인 일정 |
| --- | --- | --- |
| 출처 | 업로드한 학업 자료 | Google Calendar 또는 CatchUp 직접 입력 |
| 예시 | 마감일, 제출일, 시험일, 중요 공지 | 병원, 약속, 아르바이트, 개인 일정 |
| 수정 위치 | Upload 탭의 AI 추출 결과 수정 영역 | Google Calendar 원본 또는 Month 탭의 직접 추가 |

## 11. MVP 제외 화면 및 기능

- AI Mate를 하단 네비게이션의 개별 탭으로 제공하는 화면
- 네 번째 메인 탭
- 별도 월간 상세 계획 페이지
- 모든 LMS 자동 연동 화면
- Google Calendar 외 캘린더 연결 화면
- 친구와 공동 일정 관리 화면
- 장학금, 근로, 대외활동 추천 화면
- 정확한 완료 확률을 보여주는 예측 화면
- Today 할일 또는 주간 계획 항목을 사용자가 직접 수정하는 화면
- 같은 주 안에서 주간 계획을 여러 번 생성하는 화면
- AI Mate 계획 조정 요청을 하루에 10번 넘게 허용하는 화면

## 12. 초기 구현 이후 추가 검토 기능

- 일정 요약 캘린더에서 카테고리별 색상으로 학업 일정과 개인 일정을 구분하는 기능
- 일정 요약 캘린더에서 확인 필요 일정을 일반 일정과 다르게 표시하는 기능
- Google Calendar 외 다른 외부 캘린더 연동
- 개인 시간표 직접 연동

# Catchup Design System

## 목적

팀에서 확정한 모바일 화면 목업을 기준으로 Catchup MVP의 공통 시각 언어와 UI 컴포넌트 규칙을 정의한다. 이 문서는 반응형 웹앱 구현 시 색상, 타이포그래피, 간격, 카드, 버튼, 탭, 챗봇 UI를 일관되게 적용하기 위한 기준이다.

## 디자인 원칙

- 학업 일정의 압박감을 줄이기 위해 부드러운 라벤더 톤과 넉넉한 여백을 사용한다.
- 사용자가 1분 안에 오늘 해야 할 일을 파악할 수 있도록 화면별 핵심 행동을 상단에서 바로 드러낸다.
- AI 추천은 확정 명령이 아니라 참고용 계획으로 표현한다.
- 실제 학생 개인정보를 예시로 쓰지 않고, 모든 화면 예시는 익명화된 샘플 자료만 사용한다.
- MVP 범위 안에서 파일 업로드, 추출 확인/수정, Today, Month, AI Mate 조정 흐름에 집중한다.

## 브랜드 톤

Catchup은 놓친 일정을 다그치는 서비스가 아니라, 사용자가 다시 리듬을 잡도록 도와주는 학업 파트너다.

- 따뜻함: "오늘도 따라잡아볼까요?"처럼 부담을 낮추는 문장
- 명확함: "3개", "7/16 마감", "추출 완료"처럼 즉시 판단 가능한 정보
- 신뢰감: 추천 이유와 일정 근거를 함께 제공
- 가벼운 활기: 보라색 포인트, 스파클 아이콘, AI Mate 캐릭터 톤

## 타이포그래피

기본 폰트는 웹 환경의 시스템 산세리프를 사용한다. 별도 웹폰트 적용은 MVP 이후로 둔다.

| Style | 크기 | 굵기 | 용도 |
| --- | ---: | ---: | --- |
| Brand | 43 | 900 Italic | 온보딩 `Catch Up` 로고 텍스트 |
| Screen Title | 21-22 | 900 | 화면 제목, Today 인사말 |
| Sheet Title | 18 | 900 | AI Mate 바텀시트 제목 |
| Section Title | 16 | 900 | 섹션 제목 |
| Card Title | 12-16 | 700-900 | 카드 주요 텍스트. 화면별 정보 밀도에 맞춰 선택 |
| Body | 11-14 | 400-700 | 카드 설명, 안내 문장 |
| Meta | 9-12 | 400-900 | 날짜, 파일 크기, 배지, 탭 라벨 |

### 문장 규칙

- 버튼은 동사형으로 쓴다: `캘린더 연결하기`, `추출 결과 확인 및 수정`
- 상태는 완료/실패/필요를 명확히 쓴다: `추출 완료`, `확인 필요`, `연결하지 못했어요`
- AI 문장은 추천 근거 중심으로 쓴다: `4주 일정을 고려해 중요한 일부터 정리했어요.`
- 수치형 성과 예측을 쓰지 않는다: `완료 확률 38%` 같은 표현 금지

## 레이아웃

### 반응형 웹 기준

- 모바일 우선 기준 뷰포트는 `390px` 너비다.
- 데스크톱과 태블릿에서는 주요 콘텐츠를 모바일 기준 폭으로 중앙 정렬하거나, 구현 시 확정한 반응형 레이아웃으로 확장한다.
- 모바일 웹에서는 상단 `env(safe-area-inset-top)`과 하단 `env(safe-area-inset-bottom)`을 반영한다.
- 페이지 좌우 기본 패딩은 `20px`이다.
- 하단 탭바의 콘텐츠 높이는 `76px`이며, 실제 총 높이는 `calc(76px + env(safe-area-inset-bottom))`이다.
- 하단 탭바와 AI Mate 버튼이 있는 스크롤 화면은 콘텐츠 하단 패딩을 `calc(176px + env(safe-area-inset-bottom))` 이상 둔다.
- 플로팅 AI Mate 버튼은 우측 `20px`, 탭바 위 `18px`에 배치한다.
- 하단 탭바와 AI Mate 버튼은 뷰포트에 고정하고, 화면 본문만 세로 스크롤한다.

| 요소 | Radius | 규칙 |
| --- | ---: | --- |
| 일반 카드 | 14-16 | 흰 배경, 약한 그림자 또는 무그림자 |
| 강조 카드 | 18 | 연보라 배경, 보라 테두리 |

| CTA 버튼 | 15 | 보라 배경, 흰 텍스트 |

## 공통 컴포넌트

### Header

- 좌측에 화면 제목을 둔다.
- 우측에는 화면별 보조 액션 아이콘 1-2개만 둔다.
- 제목은 `Screen Title`을 사용한다.
- 아이콘 단독 버튼의 터치 영역은 최소 `44 x 44px`로 둔다.
- 시각 아이콘은 기본 `21px`이며, 화면 전용 목업에서 별도 크기를 정한 경우 해당 값을 따른다.

### Bottom Tab Bar

- 탭은 `Today`, `Month`, `Upload` 3개로 고정한다.
- 탭바 콘텐츠 높이는 `76px`, 아이콘은 `26px`, 라벨은 `12px`, 아이콘과 라벨 간격은 `6px`를 사용한다.
- 하단 Safe Area는 76px 높이에 포함하지 않고 별도로 더한다.
- 탭바는 `color.surface`를 96% 불투명도로 사용하고 상단에 `color.line` 1px 구분선을 둔다.
- 활성 탭은 `color.purple`, 비활성 탭은 `color.icon.muted`를 사용한다.
- 탭 라벨은 영어 유지가 가능하나, 발표 자료나 문서에서는 `오늘`, `월간`, `업로드` 병기를 허용한다.

### AI Mate Floating Button

- 모든 메인 탭 화면에 노출한다.
- 크기: `70 x 70px`, radius `35px`.
- 위치: right `20px`, bottom `calc(94px + env(safe-area-inset-bottom))`.
- 보라색 그라디언트 원 안에 전용 AI Mate 얼굴 캐릭터를 표시하고 텍스트는 넣지 않는다.
- 바깥에 흰색 `2px` 링을 두고, `color.purple.dark` 기준 opacity `0.22`, radius `14px`, y `6px` 그림자를 사용한다.
- 누르면 AI Mate 바텀시트를 연다.
- 접근성 라벨은 `AI Mate 열기`로 제공한다.

### Card

- 카드 안에는 한 가지 목적만 담는다.
- 할 일 카드는 체크박스, 제목, 출처, 우선순위, 마감일을 포함한다.
- 파일 카드는 파일 타입, 파일명, 메타 정보, 추출 상태, 이동 아이콘을 포함한다.
- AI 안내 카드는 스파클 아이콘, `AI Mate` 라벨, 추천 근거 요약을 포함한다.

### Badge / Pill

- 짧은 상태만 담는다.
- 예: `3개`, `추출 완료`, `확인 필요`, `7/16 마감`, `중간고사`
- 성공 배지는 초록, 확인 필요/임박 배지는 보라 또는 노랑 계열을 사용한다.

### Upload Zone

- 점선 테두리와 업로드 아이콘을 사용한다.
- 지원 형식은 MVP에서 `PDF 또는 이미지 파일`로 표시한다.
- PDF와 이미지 파일을 모두 업로드할 수 있다.
- Upload 화면의 확정 목업 수치는 `DESIGN_SPEC.md`의 Upload 섹션을 따른다.

### Chat Bottom Sheet

- 상단에 핸들, 제목, 닫기 버튼을 둔다.
- 사용자 말풍선은 우측 정렬 + `color.purple.soft`.
- AI Mate 말풍선은 좌측 정렬 + 흰 배경 + 스파클 아이콘.
- 입력창 아래 또는 우측에 오늘 남은 조정 횟수를 표시한다. 하루 최대 10회이며, 매일 자정에 초기화된다.

## 상태 규칙

### Empty

- 비어 있는 이유와 다음 행동을 함께 제시한다.
- 예: `아직 업로드한 자료가 없어요. 강의계획서나 LMS 공지를 등록해주세요.`

### Loading

- 화면 전체를 막기보다 해당 영역에 진행 상태를 표시한다.
- 긴 작업 문구는 `학업 정보를 읽는 중입니다.`처럼 현재 처리 중인 일을 설명한다.

### Error

- 사용자가 다음에 할 수 있는 행동을 포함한다.
- 민감한 토큰, 계정 정보, 원문 개인정보를 오류 메시지에 노출하지 않는다.

### Success

- 완료 상태는 짧게 표시한다.
- 예: `추출 완료`, `Google Calendar 연결됨`, `이번 주 계획이 생성되었습니다.`

## 접근성

- 핵심 텍스트와 배경의 대비를 충분히 확보한다.
- 아이콘 단독 버튼은 접근성 라벨을 제공한다.
- 터치 타깃은 최소 `44x44px`에 가깝게 유지한다.
- 색상만으로 상태를 구분하지 않고 텍스트 배지를 함께 제공한다.

## MVP 범위 점검

- 이 디자인 시스템은 `PRD.md`의 1차 MVP 화면에만 적용한다.
- 월간 화면은 상세 월간 계획 생성이 아니라 4주 일정 참고와 일정 확인 용도다.
- AI Mate는 하루 최대 10회의 횟수 제한이 있는 계획 조정 UI로 다룬다.
- 모든 LMS 자동 연동, 친구 공동 일정, 장기 성과 예측, 정확한 완료 확률 표시는 포함하지 않는다.

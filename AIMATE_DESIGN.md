# CatchUp AI Mate 패널 디자인 명세

## 1. 문서 목적과 적용 범위

이 문서는 플로팅 AI Mate 버튼을 눌렀을 때 열리는 모바일 AI Mate 패널 목업을 개발 구현 기준으로 변환한 화면 전용 디자인 명세다. 개발자는 원본 목업 이미지를 다시 열지 않아도 이 문서만으로 목업의 정상 대화 상태와 MVP에 필요한 파생 상태를 구현할 수 있어야 한다.

이 문서의 범위는 다음과 같다.

- 전역 플로팅 버튼에서 AI Mate 패널을 여닫는 흐름
- 패널의 위치, 크기, 레이어, 색상, 타이포그래피, 간격
- 헤더, 대화 영역, 사용자·AI 메시지, 입력창의 상세 규칙
- 키보드, 스크롤, 포커스, 반응형, 접근성 동작
- 주간 계획 생성, 계획 조정, 추천 근거 질문의 UI 상태
- 주 1회 생성 제한과 하루 10회 조정 제한의 표시 방식

이 문서는 `PRD.md`, `README.md`, `IA.md`, `USER_FLOW.md`, `SCREEN_SPEC.md`를 제품 기준으로 사용하고 `DESIGN_SYSTEM.md`의 공통 토큰을 따른다. 제품 정책이 시각 목업과 충돌하면 `PRD.md`를 우선한다.

## 2. 기준과 판정 방법

### 2.1 값의 신뢰도 표기

| 표기 | 의미 |
| --- | --- |
| `확정` | 목업 또는 제품 기준 문서에서 텍스트, 존재 여부, 순서, 정책을 직접 확인할 수 있는 값 |
| `추정` | 목업 픽셀 비율을 390px 모바일 뷰포트로 환산한 권장값 또는 렌더링 근사값 |
| `구현 결정` | 정적 목업에서 확인할 수 없지만 일관된 동작과 접근성을 위해 이 문서에서 확정한 구현 규칙 |
| `확인 필요` | 브랜드 에셋처럼 현재 자료만으로 최종 값을 정할 수 없는 항목 |

첨부 PNG의 전체 크기는 `762 × 1516px`다. 이미지에는 iPhone 기기 프레임이 포함되어 있으므로 실제 앱 DOM은 프레임, Dynamic Island, 시스템 상태바, 홈 인디케이터를 그리지 않는다. 아래 수치는 기기 안쪽 앱 화면을 약 `390 × 844px` CSS 뷰포트로 환산한 값이며 렌더링과 원근감에 따라 약 `±2px` 오차를 허용한다.

### 2.2 목업이 보여주는 상태

| 항목 | 현재 상태 | 판정 |
| --- | --- | --- |
| 배경 탭 | `Upload` 활성 | 확정 |
| AI Mate | 열린 상태 | 확정 |
| 패널 형태 | 화면 아래에서 올라온 라운드 바텀시트 | 확정 |
| 대화 | 사용자 메시지 1개, AI 응답 1개 | 확정 |
| 사용자 요청 | `이번 주 계획 짜줘` | 확정 |
| AI 응답 | 업로드 자료와 캘린더를 반영해 주간 계획 생성 완료 | 확정 |
| 조정 잔여 | `3회` | 확정 |
| 입력 상태 | 비어 있음, placeholder 표시 | 확정 |
| 전송 버튼 | 보라색 활성 표현 | 확정 |
| 키보드 | 닫힘 | 확정 |
| 로딩·오류 | 표시되지 않음 | 확정 |

`조정 잔여 3회`는 서비스 전체 한도가 3회라는 뜻이 아니다. 제품 정책상 하루 최대 10회 중 현재 3회가 남았다는 상태 예시다. 주간 계획 생성 요청과 추천 근거 질문은 조정 횟수에서 차감하지 않는다.

## 3. 화면 계층과 레이어

### 3.1 위에서 아래로 보이는 구조

```text
[기존 메인 화면: Upload]
  헤더와 업로드 영역 일부

[AI Mate 모달 레이어]
  [투명한 배경 차단 영역]
  [AI Mate 바텀시트]
    드래그 핸들
    헤더
      스파클 아이콘 + AI Mate
      조정 잔여 3회
      닫기 버튼
    스크롤 가능한 대화 영역
      사용자 메시지
      AI Mate 캐릭터 + AI 메시지
    메시지 입력 컴포저

[기존 고정 하단 탭바]
  Today / Month / Upload
[하단 Safe Area]
```

### 3.2 레이어 규칙

- 바텀시트는 현재 `Today`, `Month`, `Upload` 화면 위에 전역으로 열린다. `확정`
- 목업에서는 배경을 어둡게 만드는 검은 dim이 보이지 않는다. 배경 차단 레이어는 투명하게 사용한다. `확정`
- 바텀시트는 하단 탭바 위에서 시작해 탭바 상단까지 콘텐츠를 차지한다. 탭바는 화면에 계속 보인다. `확정`
- 패널이 열린 동안 배경 본문과 탭바의 포인터·키보드 조작은 막는다. 탭 전환은 패널을 닫은 뒤 수행한다. `구현 결정`
- 패널은 modal semantics를 사용하며 배경은 접근성 트리에서 일시적으로 비활성화한다. `구현 결정`
- z-index 권장 순서는 `page 0 < tab bar 20 < modal blocker 30 < sheet 40`이다. 시트가 탭바를 물리적으로 덮지 않으므로 탭바는 보이지만 blocker가 입력을 가로챈다. `구현 결정`
- AI Mate 플로팅 버튼은 패널이 열리면 숨긴다. 목업에 버튼이 중복 노출되지 않는다. `확정`

### 4.1 모바일 기준 치수

기준 뷰포트는 `390 × 844px`이며 상·하단 Safe Area는 브라우저가 제공하는 `env()` 값을 사용한다.

| 요소 | 권장 위치·크기 | 판정 |
| --- | --- | --- |
| 패널 상단 | 뷰포트 상단에서 약 `262px` | 추정 |
| 패널 가시 높이 | 탭바 상단까지 약 `506px` | 추정 |
| 패널 너비 | `100%` | 확정 |
| 패널 최대 너비 | 모바일 앱 콘텐츠 폭과 동일, 데스크톱에서 `430px` | 구현 결정 |
| 패널 상단 반경 | `24px 24px 0 0` | 추정 |
| 드래그 핸들 | `30 × 4px`, radius `2px` | 추정 |
| 핸들 상단 여백 | `9px` | 추정 |
| 헤더 상단 | 패널 상단에서 약 `34px` | 추정 |
| 헤더 좌우 패딩 | `20px` | 추정 |
| 헤더 최소 높이 | `50px` | 추정 |
| 대화 영역 좌우 패딩 | `20px` | 추정 |
| 입력창 좌우 여백 | `20px` | 추정 |
| 입력창 높이 | `52px` | 추정 |
| 입력창 하단 | 탭바 상단에서 약 `15px` | 추정 |

패널 상단 위치는 고정 px 대신 다음처럼 뷰포트에 적응시킨다.

```css
height: min(68.8dvh, 580px);
min-height: 500px;
bottom: calc(76px + env(safe-area-inset-bottom));
```

기준 844px 화면에서는 탭바 콘텐츠와 하단 Safe Area를 제외한 영역에 맞춰 목업과 가까운 높이가 되도록 구현한다. 높이가 작은 화면에서는 헤더와 입력창을 고정하고 대화 영역만 축소·스크롤한다.

### 4.2 패널 내부 그리드

- 패널은 `grid-template-rows: auto auto minmax(0, 1fr) auto` 구조를 권장한다. `구현 결정`
- 행 순서는 `핸들 → 헤더 → 대화 스크롤 영역 → 입력 컴포저`다. `확정`
- 헤더와 컴포저는 패널 안에 고정하고 대화 영역만 세로 스크롤한다. `구현 결정`
- 대화가 짧아도 컴포저는 패널 하단에 붙는다. 목업의 큰 중앙 여백은 빈 고정 마진이 아니라 남은 가용 공간이다. `확정`

## 8. 컴포넌트 상세

### 8.1 투명 배경 차단 영역

- 패널을 제외한 기존 화면 영역을 덮는 투명 버튼 또는 backdrop이다. `구현 결정`
- 검정 dim을 넣지 않는다. `확정`
- 탭하면 패널을 닫는다. `구현 결정`
- 스크롤 체이닝을 막아 배경 페이지가 움직이지 않게 한다. `구현 결정`
- 접근성 이름은 제공하지 않고 모달 바깥의 보조 닫기 수단으로만 동작한다. `구현 결정`

### 8.2 바텀시트 컨테이너

- 배경은 아주 옅은 라벤더 반투명 표면이다. `확정`
- 상단 좌우 모서리는 약 `24px`로 둥글고 하단은 탭바에 닿아 잘린다. `확정`
- 좌우는 뷰포트 끝까지 닿으며 별도 외곽 여백이 없다. `확정`
- `role="dialog"`, `aria-modal="true"`, `aria-labelledby`로 `AI Mate` 제목을 연결한다. `구현 결정`
- 패널을 열면 기존 대화의 마지막 읽은 위치를 유지하되 새 메시지가 있으면 마지막 메시지로 이동한다. `구현 결정`

### 8.3 드래그 핸들

- 패널 상단 중앙에 `30 × 4px`의 회색 캡슐을 둔다. `확정`
- 핸들은 시각적 affordance다. MVP에서 실제 연속 드래그 리사이즈는 구현하지 않아도 된다. `구현 결정`
- 핸들 또는 상단 영역을 아래로 짧게 swipe하면 패널을 닫는 동작은 선택 사항이다. 구현한다면 `80px` 이상 이동 또는 `0.5px/ms` 이상의 하향 속도를 닫기 기준으로 사용한다. `확인 필요`
- 키보드 사용자에게는 별도의 닫기 버튼과 `Escape` 동작을 제공하므로 핸들을 포커스 가능 요소로 만들지 않는다. `구현 결정`

### 8.4 패널 헤더

헤더는 한 행이며 다음 세 그룹으로 구성한다.

```text
[스파클 아이콘][AI Mate]      [조정 잔여 3회] [닫기]
```

- 좌측 그룹은 스파클 아이콘과 제목을 `12px` 간격으로 배치한다. `추정`
- 스파클 아이콘의 시각 크기는 약 `28 × 28px`다. 큰 네 꼭짓점 별과 작은 보조 별을 함께 사용한다. `확정`
- 아이콘 전용 터치 동작은 없고 장식 요소이므로 `aria-hidden="true"`다. `구현 결정`
- 제목은 `AI Mate`, 영문 대소문자와 공백을 그대로 쓴다. `확정`
- 우측 그룹은 잔여 횟수와 닫기 버튼을 `10~12px` 간격으로 배치한다. `추정`
- 잔여 횟수는 한 줄 고정이며 `조정 잔여 {remaining}회` 형식이다. `확정`
- 닫기 아이콘은 약 `24px`의 검은 `X`이며 실제 버튼 터치 영역은 최소 `44 × 44px`다. `추정`
- 닫기 버튼 접근성 이름은 `AI Mate 닫기`다. `구현 결정`
- 헤더 하단 구분선은 사용하지 않는다. `확정`

좁은 화면에서 공간이 부족하면 제목은 유지하고 잔여 문구를 `잔여 {remaining}회`로 축약할 수 있다. 숫자만 표시하거나 아이콘만으로 의미를 전달하지 않는다. `구현 결정`

## 9. 동작과 전환

### 9.1 열기

1. 사용자가 모든 메인 탭에 노출되는 AI Mate 플로팅 버튼을 선택한다.
2. 배경 스크롤 위치를 보존하고 배경을 inert 처리한다.
3. 투명 blocker와 패널을 표시한다.
4. 패널은 아래에서 `240ms` 동안 `ease-out`으로 올라온다.
5. 모션 감소 설정이 없으면 opacity를 `0 → 1`로 함께 전환한다.
6. 초기 포커스는 입력창이 아니라 패널 제목 또는 닫기 버튼에 둔다. 모바일에서 열자마자 키보드가 튀어나오지 않게 한다.

### 9.2 닫기

다음 동작으로 닫을 수 있다.

- 헤더의 `X` 버튼 선택
- 투명 backdrop 선택
- 키보드의 `Escape`
- 구현된 경우 하향 swipe

닫기 애니메이션은 `200ms ease-in`을 권장한다. 닫힌 뒤 포커스를 원래 AI Mate 플로팅 버튼으로 돌리고 배경 스크롤 위치를 복원한다. 전송하지 않은 입력이 있더라도 같은 앱 세션에서 다시 열면 유지한다. 페이지 이탈 시 임시 입력 보존은 MVP 필수 사항이 아니다.

### 9.3 입력과 AI 응답

```text
입력 가능
-> 사용자 전송
-> 사용자 말풍선을 즉시 추가
-> 입력창 비우기
-> AI 응답 생성 중 표시
-> 성공 시 AI 메시지 추가 및 관련 화면 데이터 갱신
-> 실패 시 재시도 UI 표시
```

- 응답 중에는 입력창을 완전히 잠그지 않는다. 단, 같은 메시지의 중복 전송은 막는다. `구현 결정`
- AI 응답 생성 중에는 캐릭터 옆 흰 말풍선에 `답변을 준비하고 있어요`와 점 3개 애니메이션을 표시한다. `구현 결정`
- 계획 생성·조정처럼 중복 실행 위험이 있는 요청은 진행 중 추가 실행을 막고 `계획을 반영하고 있어요`를 표시한다. `구현 결정`
- 성공 후 Today와 Month의 관련 데이터를 다시 가져오고, AI 응답에 완료 결과와 추천 근거를 표시한다. `확정`
- API 오류 시 기존 계획은 바꾸지 않으며 `요청을 처리하지 못했어요. 잠시 후 다시 시도해주세요.`와 `다시 시도`를 표시한다. `구현 결정`

## 10. 요청 유형과 횟수 정책

### 10.1 요청 분류

| 요청 유형 | 예시 | 조정 횟수 차감 | 결과 |
| --- | --- | ---: | --- |
| 주간 계획 생성 | `이번 주 계획 짜줘` | 차감 안 함 | 월~일 주간 계획 생성 |
| 계획수립 요구사항 | `일요일에는 쉬는 시간을 많이 확보해줘` | 생성 전이면 차감 안 함 | 생성 요청 조건으로 반영 |
| 오늘 계획 조정 | `오늘 할 일을 줄여줘` | 성공 시 1회 | 기존 계획 조정 |
| 이번 주 계획 조정 | `수요일 할 일을 목요일로 옮겨줘` | 성공 시 1회 | 기존 계획 조정 |
| 추천 근거 질문 | `왜 이 과제를 오늘 해야 해?` | 차감 안 함 | 근거 설명 |
| 일반 도움말 | `어떤 자료를 올릴 수 있어?` | 차감 안 함 | 안내 응답 |

서버가 요청 유형을 최종 판정한다. 조정 요청은 성공적으로 계획 변경이 반영된 경우에만 사용량을 증가시키며, 네트워크 오류·AI 처리 실패·단순 질문은 차감하지 않는다. 동시에 여러 기기에서 요청하더라도 서버 원자 연산으로 하루 최대 10회를 넘지 않게 한다. `구현 결정`

### 10.2 헤더의 잔여 횟수

- 항상 `조정 잔여 {0~10}회`로 표시한다. `확정`
- 값은 로컬 추측값이 아니라 서버 응답을 기준으로 한다. `구현 결정`
- 조정 성공 직후 즉시 1 감소한다. `확정`
- 매일 사용자 로컬 시간대 자정에 10회로 초기화한다. 기준 시간대는 사용자 설정값을 사용하고 미설정 시 서비스 기본 시간대 정책을 따른다. `확정` 및 `구현 결정`
- 3회 이하에서는 문구를 유지하되 별도 경고색으로 압박하지 않는다. `구현 결정`
- 0회가 되면 잔여 문구를 오류색이 아닌 보조색으로 표시하고, 조정 의도로 판정된 요청의 전송을 서버에서 차단한다. 일반 질문과 추천 근거 질문은 계속 허용한다. `구현 결정`

### 10.3 주간 계획 생성 제한

- 생성 범위는 월요일부터 일요일까지다. `확정`
- 사용자가 설정한 요일과 시간에 주 1회만 생성할 수 있다. `확정`
- 같은 주의 계획이 이미 있으면 새 계획 생성 요청을 실행하지 않는다. `확정`
- 이 경우 AI는 다음과 같이 안내한다.

```text
이번 주 계획은 이미 생성했어요.
변경할 내용이 있다면 원하는 방식으로 조정을 요청해주세요.
다음 계획은 {다음 생성 가능 일시}에 만들 수 있어요.
```

- 생성 가능 시각 전 요청에는 설정 요일·시간과 다음 가능 시각을 알려준다. `확정`
- 업로드 자료가 없으면 Upload 탭으로 이동하는 링크를 제공하되 자동 이동하지 않는다. `구현 결정`
- Google Calendar가 연결되지 않았으면 연결 필요를 알리고 연결 화면으로 가는 명시적 버튼을 제공한다. `확정`

## 11. 필수 화면 상태

### 11.1 정상 대화 상태

목업과 동일한 기본 상태다. 사용자와 AI 메시지가 대화 순서대로 보이고, 헤더에 잔여 횟수, 하단에 입력창이 표시된다.

### 11.2 빈 대화 상태

- 캐릭터와 짧은 인사말을 대화 영역 상단에 표시한다.
- 권장 문구: `무엇을 같이 정리해볼까요? 이번 주 계획 생성이나 계획 조정을 요청할 수 있어요.`
- 빠른 질문 칩은 MVP 구현을 단순화하기 위해 필수로 두지 않는다. `구현 결정`

### 11.3 응답 생성 중

- 사용자 말풍선은 즉시 표시한다.
- AI 위치에 `답변을 준비하고 있어요` 로딩 말풍선을 표시한다.
- 계획 데이터 변경은 성공 응답이 올 때 한 번만 반영한다.
- 스크린 리더에는 `AI Mate가 답변을 준비하고 있습니다`를 polite live region으로 알린다.

### 11.4 조정 잔여 0회

- 헤더는 `조정 잔여 0회`로 표시한다.
- 계획 조정 요청을 보내면 사용자 메시지는 기록하되 계획 변경은 실행하지 않고 제한 안내를 응답한다.
- 안내 문구: `오늘 가능한 계획 조정을 모두 사용했어요. 내일 다시 조정할 수 있어요. 추천 이유를 묻거나 현재 계획을 확인하는 대화는 계속할 수 있어요.`
- 입력창 전체를 비활성화하지 않는다. 질문은 계속 가능해야 한다.

### 11.5 같은 주 생성 완료

- 재생성 요청은 차단한다.
- 기존 계획을 삭제하거나 덮어쓰지 않는다.
- AI 응답에서 `조정 요청`으로 명확하게 유도한다.
- 이 차단은 조정 횟수를 차감하지 않는다.

### 11.6 전제 데이터 부족

| 부족한 정보 | 안내 | 다음 행동 |
| --- | --- | --- |
| 업로드 자료 없음 | 학업 자료가 필요함을 설명 | `Upload로 이동` |
| Google Calendar 미연결 | 개인 일정 반영을 위해 연결 필요 | `캘린더 연결하기` |
| 추출 확인 필요 | 불확실한 항목이 있음을 설명 | `추출 결과 확인` |
| 생성 요일·시간 미설정 | 생성 주기 설정 필요 | `생성 일정 설정` |

버튼이 메시지 안에 추가될 경우 흰 AI 말풍선 아래에 텍스트 버튼 또는 보라색 outline 버튼으로 배치한다. 버튼 선택은 사용자의 명시적 행동이며 AI가 계정 연결이나 추출 정보 수정을 대신하지 않는다.

### 11.7 네트워크·서버 오류

- 마지막 사용자 메시지 아래에 실패 상태를 표시한다.
- 오류 전문, API 키, OAuth 토큰, 업로드 원문, 개인정보를 노출하지 않는다.
- `다시 시도`는 동일 idempotency key를 재사용해 계획이 중복 생성·조정되지 않게 한다.
- 재시도 실패가 반복돼도 기존 계획과 잔여 횟수는 서버 확정값 없이 변경하지 않는다.

## 12. 키보드와 모바일 뷰포트

- `100vh` 대신 `100dvh`를 우선 사용해 모바일 브라우저 주소창과 가상 키보드 변화에 대응한다. `구현 결정`
- 입력창 포커스 시 패널의 상단 위치는 유지하되 컴포저가 키보드 위에 오도록 visual viewport에 맞춘다. 공간이 부족하면 대화 영역 높이만 줄인다. `구현 결정`
- iOS에서는 컴포저 하단 패딩에 `env(safe-area-inset-bottom)`을 적용하되, 키보드가 열린 동안 중복 inset이 생기지 않게 한다. `구현 결정`
- 키보드가 열린 상태에서 새 메시지를 보내면 최신 메시지가 컴포저에 가리지 않도록 `scroll-padding-bottom`을 적용한다. `구현 결정`
- 패널 내부 스크롤의 overscroll이 배경 페이지로 전달되지 않도록 `overscroll-behavior: contain`을 사용한다. `구현 결정`
- 입력 중 X를 눌러 닫아도 작성 중 텍스트를 같은 세션 동안 유지한다. `구현 결정`

## 13. 반응형 규칙

### 13.1 모바일 `320~430px`

- 패널 너비는 뷰포트 전체다.
- 좌우 패딩은 기본 `20px`, 360px 미만에서는 `16px`로 줄인다.
- 메시지 최대 너비는 `76%`를 유지하되 AI 캐릭터 포함 총 너비가 영역을 넘지 않게 한다.
- 헤더는 한 줄 유지한다. 320px 근처에서 잔여 문구만 `잔여 n회`로 축약할 수 있다.
- 탭바 상단을 패널의 하단 기준으로 사용한다.

### 13.2 태블릿·데스크톱 `431px 이상`

- AI Mate는 현재 화면 맥락을 유지하는 전역 보조 패널이므로 최대 너비 `430px`로 제한하고 메인 앱 컨테이너의 오른쪽 아래에 정렬한다. `구현 결정`
- 모바일 앱 자체가 `430px` 폭으로 중앙 정렬된 구조라면 패널도 그 컨테이너 너비를 그대로 사용한다.
- 패널 상단 반경은 유지하고 좌우가 브라우저 끝에 닿지 않는 경우 하단 모서리도 `24px`로 둥글게 할 수 있다. `구현 결정`
- 데스크톱에서는 투명 backdrop을 앱 컨테이너 범위에만 적용한다. `구현 결정`

## 14. 접근성

- 모든 본문 텍스트와 배경은 WCAG AA 대비를 목표로 한다. 연보라 배경 위 본문은 `#111014`를 사용한다.
- 닫기와 전송 버튼은 최소 `44 × 44px` 터치 영역과 명확한 접근성 이름을 갖는다.
- 패널이 열리면 포커스를 패널 안에 가두고 닫히면 원래 플로팅 버튼으로 복원한다.
- 대화 목록은 `role="log"`, `aria-live="polite"`, `aria-relevant="additions text"`를 권장한다.
- 기존 메시지 전체를 패널이 열릴 때마다 읽지 않고 새 AI 응답만 알린다.
- 잔여 횟수 변경은 `조정 가능 횟수 2회 남음`처럼 스크린 리더에 한 번 알린다.
- 캐릭터와 스파클은 장식 요소이므로 대체 텍스트를 중복 제공하지 않는다. 발화자는 메시지의 접근성 레이블에서 `AI Mate`로 구분한다.
- 색상만으로 사용자·AI·오류·제한 상태를 구분하지 않는다. 정렬, 발화자 이름 또는 문구를 함께 사용한다.
- `prefers-reduced-motion: reduce`에서는 시트와 로딩 점 애니메이션을 즉시 전환 또는 짧은 fade로 대체한다.

## 15. 데이터와 UI 연결

### 15.1 패널에 필요한 최소 상태

```text
isOpen
activeTab
messages[]
draftMessage
requestStatus: idle | sending | generating | success | error
adjustmentRemainingToday: 0..10
weeklyPlanStatus: not-created | available | generated | blocked
nextWeeklyGenerationAt
calendarConnectionStatus
uploadSummary
```

### 15.2 메시지 최소 필드

```text
id
role: user | assistant | system
text
createdAt
status: sending | sent | failed
intent: generate-plan | adjust-plan | explain-reason | help | unknown
actionResult?: generated | adjusted | blocked | failed
recommendationReasons?: string[]
remainingAdjustmentCount?: number
actions?: { label, destination }[]
```

- 채팅 UI에 실제 학생 이름, 학번, 이메일, 성적, 캘린더 원문 같은 민감정보를 불필요하게 노출하지 않는다.
- 업로드 자료의 전체 원문보다 계획에 필요한 추출·확인된 정보만 응답에 요약한다.
- 클라이언트가 요청 의도를 추정해 횟수를 임의 차감하지 않는다. 서버 결과의 `remainingAdjustmentCount`를 표시한다.
- 계획 생성·조정 성공 후 관련 캐시를 무효화해 Today와 Month가 최신 상태를 보여주게 한다.

## 16. MVP에서 제외하는 UI와 동작

- AI Mate를 네 번째 하단 탭으로 추가하지 않는다.
- 같은 주에 주간 계획을 다시 생성하거나 기존 계획을 무단 덮어쓰지 않는다.
- 하루 10회를 넘는 계획 조정을 허용하지 않는다.
- 사용자가 계획 항목의 시간, 날짜, 우선순위, 내용을 폼으로 직접 수정하는 UI를 패널에 넣지 않는다.
- AI Mate가 Upload의 추출 필드를 대신 수정하지 않는다.
- 전체 월간 세부 계획 생성, 다른 외부 캘린더 연결, LMS 완전 자동 연동을 제안하지 않는다.
- 정확한 완료 확률, 성적 예측, 장기 학습 성과 예측을 메시지에 표시하지 않는다.
- 음성 입력, 파일 첨부, 메시지 편집·삭제, 이모지 반응, 다중 대화방, 대화 검색은 MVP에서 제외한다.
- 기기 프레임, Dynamic Island, 상태바, 홈 인디케이터를 앱 UI로 그리지 않는다.

## 17. 구현 체크리스트

### 레이아웃

- [ ] 패널이 모든 메인 탭에서 같은 위치와 크기로 열린다.
- [ ] 기준 390px 화면에서 패널 상단이 약 262px 지점에 온다.
- [ ] 패널 상단 모서리는 약 24px로 둥글고 드래그 핸들이 중앙에 있다.
- [ ] 헤더와 컴포저는 고정되고 대화 영역만 스크롤된다.
- [ ] 입력창은 탭바 위 약 15px 간격을 유지한다.
- [ ] 짧은 대화의 중앙 여백이 가용 공간으로 자연스럽게 늘어난다.

### 시각

- [ ] 패널은 거의 흰 라벤더 반투명 표면이며 검은 dim을 사용하지 않는다.
- [ ] 사용자 말풍선은 우측 연보라, AI 말풍선은 좌측 흰색이다.
- [ ] AI 캐릭터가 말풍선 바깥 왼쪽에 표시된다.
- [ ] 헤더의 스파클, 제목, 잔여 횟수, X 순서가 목업과 같다.
- [ ] 입력창 테두리, placeholder, 보라색 전송 아이콘이 표시된다.

### 동작

- [ ] 플로팅 버튼, X, backdrop, Escape로 정해진 열기·닫기 동작이 된다.
- [ ] 패널이 열려 있을 때 배경 스크롤과 탭 조작이 차단된다.
- [ ] 전송, 로딩, 성공, 실패, 재시도가 중복 실행 없이 동작한다.
- [ ] 키보드가 열려도 입력창과 마지막 메시지가 가려지지 않는다.
- [ ] 패널을 닫고 다시 열 때 세션 내 draft와 대화 위치가 합리적으로 유지된다.

### 제품 정책

- [ ] 주간 계획 생성은 설정 요일·시간에 주 1회만 가능하다.
- [ ] 같은 주 재생성 요청은 차단하고 AI 조정으로 안내한다.
- [ ] 계획 조정만 성공 시 하루 사용량이 1회 차감된다.
- [ ] 잔여 횟수는 0~10 범위의 서버 값으로 표시된다.
- [ ] 잔여 0회에도 추천 근거 질문과 일반 대화는 가능하다.
- [ ] 계획 생성·조정 결과에 간단한 추천 근거가 포함된다.
- [ ] 수치형 완료 확률이나 성과 예측을 표시하지 않는다.

### 접근성·보안

- [ ] dialog, focus trap, focus restore, Escape 동작이 구현된다.
- [ ] 아이콘 버튼의 터치 영역이 최소 44px이고 접근성 이름이 있다.
- [ ] 새 AI 메시지와 횟수 변경이 과도하지 않게 안내된다.
- [ ] 모션 감소 설정을 존중한다.
- [ ] 오류 메시지와 대화 UI에 토큰·키·민감 개인정보를 노출하지 않는다.

## 18. QA 기준 시나리오

1. `Upload`에서 AI Mate를 열면 목업과 같은 패널, 헤더, 빈 입력창이 보인다.
2. `이번 주 계획 짜줘`를 보내면 사용자 말풍선, 로딩, AI 응답 순서로 표시된다.
3. 생성 가능한 시각이고 전제 데이터가 갖춰졌다면 월~일 계획이 한 번 생성되고 Today·Month에 반영된다.
4. 같은 주에 다시 생성하면 기존 계획은 유지되고 조정 요청 안내가 표시된다.
5. `오늘 할 일을 줄여줘`가 성공하면 잔여 횟수가 정확히 1 감소한다.
6. 추천 이유 질문과 실패한 요청은 잔여 횟수를 줄이지 않는다.
7. 잔여 0회에서 조정은 차단되지만 일반 질문 입력은 가능하다.
8. 320px 폭, 390px 폭, 큰 글자, 화면 회전, 키보드 열린 상태에서 가로 잘림이 없다.
9. 스크린 리더와 키보드만으로 열기, 메시지 입력, 전송, 닫기가 가능하다.
10. 네트워크 재시도에도 계획이 중복 생성되거나 잔여 횟수가 이중 차감되지 않는다.

## 19. 최종 확인이 필요한 에셋

다음 항목은 레이아웃 구현을 막지는 않지만 시각 완성도를 위해 디자인 팀의 원본 제공이 필요하다.

- AI Mate 캐릭터 SVG 또는 투명 배경 고해상도 PNG
- 헤더 스파클 전용 아이콘이 있는 경우 해당 SVG
- 사용자 말풍선과 캐릭터의 정확한 그라디언트 정지점
- 바텀시트 drag-to-dismiss를 MVP에 포함할지 여부

에셋이 제공되기 전에는 `DESIGN_SYSTEM.md`의 보라색 토큰과 동등한 오픈소스 아이콘을 사용하되, 캐릭터를 일반 로봇·챗 아이콘으로 대체하지 않는다.

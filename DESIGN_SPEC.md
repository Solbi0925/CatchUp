# Catchup Design Spec

## 목적

팀에서 확정한 화면별 목업을 기준으로 수치, 컴포넌트 크기, 간격, 상태 표현을 더 촘촘하게 정의한다. 이 문서는 `DESIGN_SYSTEM.md`의 공통 규칙을 반응형 웹앱 화면에 적용하기 위한 픽셀 보정 기준이다.

## 기준

- 플랫폼: 반응형 웹앱
- 기준 화면: 모바일 우선 `390px` 너비 뷰포트
- 기준 탭: `Today / Month / Upload`
- 좌우 화면 패딩: `20`
- 메인 화면 상단 패딩: `24`
- 탭바 콘텐츠 높이: `76` + 하단 Safe Area
- 탭바가 있는 화면의 콘텐츠 하단 패딩: `calc(176px + env(safe-area-inset-bottom))` 이상
- AI Mate 플로팅 버튼 위치: 우측 `20`, 하단 `calc(94px + env(safe-area-inset-bottom))`
- 상단 Safe Area: `env(safe-area-inset-top)` 반영

## 구현 체크리스트

- Onboarding은 첫 화면에서 `Catch Up`이 가장 큰 시각 신호여야 한다.
- Calendar 연결 화면은 흰 패널 안에 일러스트, 혜택, CTA가 한 덩어리로 보여야 한다.
- Today는 사용자가 스크롤 없이 추천 카드와 오늘 할 일 시작점을 볼 수 있어야 한다.
- Today는 완료 체크와 추천 근거 확인만 제공하고, 할 일 추가·수정 UI를 노출하지 않는다.
- Month는 월간 계획 자동 생성처럼 보이지 않게, 일정 밀도와 주요 일정 표시 중심으로 둔다.
- Month는 날짜 상세 바텀시트에서 일정 추가/수정이 가능해야 한다.
- Upload는 업로드 박스와 업로드한 자료, 추출 확인 카드가 한 흐름으로 읽혀야 한다.
- AI Mate는 계획 조정 UI이며, 오늘 남은 조정 횟수와 하루 최대 10회 제한을 항상 보여준다.
- 실제 학생 개인정보나 실제 학교 계정 정보는 화면 예시에 넣지 않는다.
- 추천 문구에 정확한 완료 확률이나 성과 예측 수치를 쓰지 않는다.

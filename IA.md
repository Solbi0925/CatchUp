# CatchUp IA

## 1. IA 목표

CatchUp의 MVP 정보 구조는 사용자가 학업 자료와 Google Calendar 개인 일정을 연결한 뒤, 오늘 해야 할 학습 항목, 주간/월간 일정, 학업 자료 업로드를 하단 네비게이션으로 명확히 나누어 사용할 수 있도록 구성한다.

AI 챗봇은 하단 네비게이션의 개별 탭으로 두지 않고, 어떤 화면에 있어도 접근할 수 있는 전역 플로팅 탭으로 제공한다. 사용자는 자료 업로드 화면에 올린 학업 자료와 Google Calendar 개인 일정을 바탕으로 AI 챗봇에게 주간 계획 생성을 요청하고, 계획 생성 시 요청사항을 자연어로 입력할 수 있다. 계획 생성 후 수정하고 싶은 부분이 있으면 같은 플로팅 AI 챗봇을 통해 계획 조정을 요청한다.

## 2. 전체 구조

| 구분 | 화면 이름 | 핵심 목적 |
| --- | --- | --- |
| 온보딩 | 서비스 소개/시작 | CatchUp의 핵심 사용 흐름을 간단히 안내하고 시작한다. |
| 온보딩 | Google Calendar 연동 | 개인 일정을 불러와 학업 계획 생성에 반영할 수 있도록 연결한다. |
| 메인 탭 1 | 오늘의 할일 | AI가 이번 주 계획에서 오늘로 배치한 학습 항목을 빠르게 확인한다. |
| 메인 탭 2 | 주간,월간 일정 | 학업 일정과 개인 일정을 주간/월간 단위로 확인한다. |
| 메인 탭 3 | 자료 업로드 | 학업 자료를 업로드하고 AI 추출 결과를 확인 및 수정한다. |
| 전역 플로팅 | AI 챗봇 | 어떤 화면에서든 주간 계획 생성, 일일/주간 계획 조정, 추천 근거 질문을 처리한다. |

## 3. 이동 방식

### 3.1 첫 진입 흐름

```text
온보딩
-> Google Calendar 연동
-> 메인 앱
-> 오늘의 할일
```

- 첫 사용자는 온보딩을 본 뒤 Google Calendar 연동 화면으로 이동한다.
- Google Calendar 연동이 완료되면 메인 앱으로 진입한다.
- 메인 앱의 기본 진입 탭은 `오늘의 할일`이다.

### 3.2 메인 네비게이션

메인 앱은 하단 네비게이션 바를 사용한다.

```text
CatchUp
├─ 오늘의 할일
├─ 주간,월간 일정
└─ 자료 업로드
```

- MVP에서는 하단 네비게이션의 3개 탭을 핵심 구조로 사용한다.
- `AI 챗봇`은 하단 네비게이션 탭이 아니라 전역 플로팅 탭으로 제공한다.
- 플로팅 AI 챗봇은 `오늘의 할일`, `주간,월간 일정`, `자료 업로드` 화면 어디에서든 열 수 있다.
- 이전 IA의 가로 스크롤 페이지 이동은 사용하지 않는다.
- 각 탭은 독립된 주요 목적을 가지며, MVP 범위를 벗어나는 별도 페이지는 만들지 않는다.

### 3.3 전역 플로팅 AI 챗봇

```text
모든 메인 화면
└─ 플로팅 AI 챗봇 버튼
   └─ AI 챗봇 패널
```

- 플로팅 버튼은 메인 앱의 모든 탭 위에 항상 접근 가능하게 둔다.
- 사용자가 버튼을 누르면 AI 챗봇 패널이 열린다.
- 챗봇 패널은 현재 화면 맥락과 사용자가 업로드한 자료, Google Calendar 일정, 생성된 주간 계획을 함께 참고한다.
- 주간 계획 생성, 계획수립 요청사항 입력, 계획 생성 후 수정 요청, 추천 근거 질문은 모두 플로팅 AI 챗봇에서 처리한다.

## 4. 온보딩 IA

### 4.1 온보딩

#### 목적

사용자가 CatchUp이 어떤 문제를 해결하는 앱인지 빠르게 이해하고 시작하도록 한다.

#### 포함 정보

- 앱 이름: CatchUp
- 한 줄 설명
  - 강의계획서, LMS 공지, 과제, 시험, Google Calendar 일정을 바탕으로 오늘과 이번 주에 해야 할 일을 추천해주는 학업 일정 매니저
- 핵심 기능 요약
  - 학업 자료 업로드
  - AI 일정 추출
  - Google Calendar 기반 주간 계획 생성
  - 오늘의 할일 확인
  - 플로팅 AI 챗봇 기반 계획 생성 및 조정
- 시작 버튼

#### 주요 행동

- 서비스 핵심 가치 확인
- 시작 버튼 선택
- Google Calendar 연동 화면으로 이동

### 4.2 Google Calendar 연동 화면

#### 목적

사용자의 개인 일정을 불러와 학업 일정과 함께 계획 생성에 반영할 수 있도록 한다.

#### 포함 정보

- Google Calendar 연동 안내
- 연동 필요 이유
  - 개인 일정과 학업 일정 충돌 방지
  - 오늘의 일정 확인
  - 주간 계획 생성 시 개인 일정 반영
- Google Calendar 연결 버튼
- 연결 상태
  - 연결 전
  - 연결 중
  - 연결 완료
  - 연결 실패
- 연결 실패 시 재시도 안내

#### 주요 행동

- Google Calendar 연결
- 연결 권한 승인
- 연결 완료 후 메인 앱 진입

#### MVP 제한

- Google Calendar 외 다른 외부 캘린더는 연결하지 않는다.
- 개인 시간표 직접 연동은 포함하지 않는다.
- 실제 민감 일정은 저장하지 않고, 필요한 일정 정보만 계획 생성에 사용한다.

## 5. 메인 탭별 IA

### 5.1 탭 1: 오늘의 할일

#### 목적

사용자가 앱을 열었을 때 AI가 이번 주 계획에서 오늘로 배치한 학습 항목을 빠르게 확인하도록 한다.

#### 포함 정보

- 오늘 날짜
- 오늘의 할일 목록
  - AI가 앞으로 4주간의 일정, 난이도, 마감일, 우선순위, 예상 소요시간, Google Calendar 개인 일정을 고려해 주간 계획을 수립했을 때 오늘로 배치한 학습 항목
  - 과제 진행
  - 시험 공부
  - 수업 준비
  - 복습 또는 예습
- 각 할일의 기본 정보
  - 과목명 또는 자료 출처
  - 할일 제목
  - 예상 소요시간
  - 우선순위
  - 완료 여부
  - 추천 근거 요약
- 오늘 예정 일정 요약
  - 오늘 마감인 과제
  - 오늘 제출해야 하는 항목
  - 오늘 예정된 시험
  - Google Calendar 개인 일정
- 플로팅 AI 챗봇 버튼

#### 주요 행동

- 오늘의 할일 확인
- 오늘 예정 일정 요약 확인
- 완료 체크
- 추천 근거 확인
- 계획 변경이 필요하면 플로팅 AI 챗봇을 열어 조정 요청

#### MVP 제한

- 사용자가 오늘 할일의 시간, 우선순위, 내용을 직접 수정하는 기능은 초기 구현에서 제외한다.
- 주간 계획 생성 또는 조정 입력은 화면 본문에 따로 두지 않고, 플로팅 AI 챗봇에서 처리한다.

### 5.2 탭 2: 주간,월간 일정

#### 목적

학업 중요 일정과 Google Calendar 개인 일정을 주간/월간 단위로 확인하고, AI가 생성한 이번 주 계획을 요일별로 확인하도록 한다.

#### 포함 정보

- 월간 일정 요약 캘린더
  - 일반 달력처럼 날짜를 숫자로 표시
  - 일정이 있는 날짜는 숫자 아래 점 개수로 표시
  - AI가 학업 자료에서 추출한 마감일
  - 제출일
  - 시험일
  - 중요 공지 일정
  - Google Calendar 개인 일정
  - 날짜 선택 시 일정 상세 요약 팝업 표시
- 주간 플래너
  - 월요일부터 일요일까지 요일별 계획
  - 각 요일의 학습 항목
  - 개인 일정과 학습 일정의 충돌 여부
  - 미완료 항목의 재배치 결과
- 일정 상세 팝업
  - 선택 날짜
  - 일정명
  - 유형: 마감, 제출, 시험, 중요 공지, 개인 일정
  - 출처: 업로드 자료 또는 Google Calendar
- 플로팅 AI 챗봇 버튼

#### 주요 행동

- 월간 캘린더에서 일정이 있는 날짜 확인
- 날짜를 선택해 해당 날짜의 일정 상세 팝업 확인
- 이번 주 계획을 요일별로 확인
- 완료한 일 체크
- 추천 근거 확인
- 계획 변경이 필요하면 플로팅 AI 챗봇을 열어 조정 요청

#### MVP 제한

- 월간 화면은 일정 확인용 캘린더이며, 전체 월간 계획을 세부 단위까지 자동 생성하지 않는다.
- Google Calendar 외 다른 외부 캘린더는 연결하지 않는다.
- 사용자가 주간 계획 항목의 시간, 날짜, 우선순위, 내용을 직접 수정하는 기능은 초기 구현에서 제외한다.
- 캘린더 점 색상으로 일정 유형을 구분하는 기능과 확인 필요 일정을 다르게 표시하는 기능은 초기 구현에서 제외한다.

### 5.3 탭 3: 자료 업로드

#### 목적

사용자가 강의계획서, LMS 공지, 과제 명세서 등 학업 자료를 업로드하고, AI가 추출한 정보를 직접 확인 및 수정하도록 한다.

업로드한 자료와 사용자가 수정한 추출 정보는 플로팅 AI 챗봇이 주간 계획을 생성하거나 계획수립 요청사항을 반영할 때 사용하는 핵심 근거가 된다.

#### 포함 정보

- 학업 자료 업로드 영역
  - MVP에서는 PDF 또는 이미지 중 하나의 파일 형식만 지원
  - 강의계획서
  - LMS 공지
  - 과제 명세서
- 업로드한 자료 목록
  - 파일명
  - 업로드 날짜
  - 추출 상태
  - 확인 필요 여부
- AI 추출 결과 확인 및 수정 영역
  - AI가 추출을 완료한 뒤 추출 정보를 사용자에게 표시
  - 사용자가 틀린 부분을 직접 입력해 수정
  - 확인 필요 항목 표시
  - 과제명
  - 마감일
  - 시험일
  - 제출 방식
  - 준비물
  - 중요 공지
  - 수업 일정 정보
- 플로팅 AI 챗봇 버튼

#### 주요 행동

- 학업 자료 업로드
- 업로드한 자료 목록 확인
- AI 추출 정보 확인
- 잘못 추출된 정보 직접 수정
- 확인 필요 항목 검토
- 수정된 추출 정보 저장
- 업로드한 자료를 바탕으로 플로팅 AI 챗봇에 주간 계획 생성 요청
- 플로팅 AI 챗봇에 계획수립 요청사항 입력

#### MVP 제한

- 모든 파일 형식을 동시에 지원하지 않는다.
- 모든 LMS와 학교 시스템을 자동 연동하지 않는다.
- 추출 정보 수정은 AI 챗봇이 대신 처리하지 않고, 사용자가 직접 입력해 수정한다.

## 6. 전역 플로팅 AI 챗봇 IA

### 목적

사용자가 어떤 화면에 있든 AI 챗봇을 열어 주간 계획 생성, 계획수립 요청사항 입력, 생성된 계획 조정, 추천 근거 질문을 할 수 있도록 한다.

### 포함 정보

- 플로팅 AI 챗봇 버튼
  - 모든 메인 탭에서 접근 가능
  - 화면 오른쪽 하단 또는 하단 네비게이션 위에 배치
- AI 챗봇 패널
  - 대화 영역
  - 자연어 입력 영역
  - 주간 계획 생성 설정 요약
  - 이번 주 주간 계획 생성 완료 여부
  - 이번 주 AI 챗봇 계획 조정 잔여 횟수
- 주간 계획 생성 설정
  - 사용자가 정한 주간 계획 생성 요일
  - 사용자가 정한 주간 계획 생성 시간
  - 다음 주간 계획 생성 예정 시각
  - 주간 계획 생성 가능 여부
- AI가 참고하는 정보
  - 자료 업로드 화면에 업로드된 학업 자료
  - 사용자가 직접 수정한 AI 추출 결과
  - Google Calendar 개인 일정
  - 완료한 일과 미완료 항목
  - 사용자의 자연어 요청사항

### 주요 행동

- 주간 계획 생성 요일과 시간 설정
- 업로드 자료 기반 주간 계획 생성 요청
- 계획 생성 시 자연어 요청사항 입력
- 생성된 계획에서 수정하고 싶은 부분을 자연어로 요청
- 오늘 또는 이번 주 계획 조정 요청
- 추천 이유 질문
- 이번 주 계획 조정 잔여 횟수 확인

### 계획 생성 요청 예시

- "이번 주 계획 짜줘."
- "자료 업로드 화면에 올린 강의계획서랑 과제 명세서 기반으로 이번 주 계획 짜줘."
- "일요일에는 쉬는 시간을 많이 확보해줘."
- "매일 15분 이상 영어 공부를 넣어줘."
- "팀플 회의가 많으니 과제는 짧게 나눠서 배치해줘."

### 계획 조정 요청 예시

- "오늘 할 일을 조금 줄여줘."
- "마감이 가까운 과제부터 하고 싶어."
- "시험 공부 시간을 늘려줘."
- "수요일에 개인 일정이 많으니 그날 할 일을 줄여줘."
- "이 계획에서 알고리즘 과제 시간을 더 앞당겨줘."

### 추천 근거 요소

- 마감일까지 남은 기간
- 예상 소요시간
- 중요도
- Google Calendar 개인 일정과의 충돌 여부
- 시험과의 연관성

### MVP 제한

- AI 챗봇은 하단 네비게이션의 개별 탭으로 만들지 않는다.
- 주간 계획 생성은 사용자가 설정한 요일과 시간에 주 1회만 가능하다.
- 한 주의 계획 범위는 월요일부터 일요일까지로 한다.
- AI 챗봇을 통한 일일/주간 계획 조정 요청은 1주에 최대 3번만 가능하다.
- 사용자가 직접 계획 항목을 손으로 수정하는 기능은 초기 구현에서 제외한다.
- 정확한 완료 확률 같은 수치형 예측은 제공하지 않는다.

## 7. 초기 구현 이후 추가 검토 기능

아래 기능은 있으면 좋지만 10주 MVP 초기 구현 범위에서는 제외하고, 시간이 남을 때 추가로 검토한다.

- 사용자가 오늘 할일 또는 주간 계획 항목의 시간, 날짜, 우선순위, 내용을 직접 수정하는 기능
- 일정 요약 캘린더에서 점 색상으로 학업 일정과 개인 일정을 구분하는 기능
- 일정 요약 캘린더에서 확인 필요 일정을 일반 일정과 다르게 표시하는 기능
- 같은 주 안에서 주간 계획을 여러 번 생성하는 기능
- AI 챗봇 계획 조정 요청을 1주에 3번 넘게 허용하는 기능
- Google Calendar 외 다른 외부 캘린더 연동
- 개인 시간표 직접 연동

## 8. MVP 범위 자체 점검

| 점검 항목 | 결과 | 근거 |
| --- | --- | --- |
| 온보딩과 Google Calendar 연동 흐름이 있는가? | 적합 | 첫 진입 흐름을 온보딩 -> Google Calendar 연동 -> 메인 앱으로 정의했다. |
| 메인 네비게이션이 3개 탭으로 정리되었는가? | 적합 | 오늘의 할일, 주간/월간 일정, 자료 업로드 3개 탭으로 정의했다. |
| AI 챗봇이 개별 탭이 아닌 플로팅 탭인가? | 적합 | 모든 메인 화면에서 접근 가능한 전역 플로팅 AI 챗봇으로 정의했다. |
| 각 탭의 목적과 포함 정보가 있는가? | 적합 | 모든 탭에 목적, 포함 정보, 주요 행동, MVP 제한을 명시했다. |
| 자료 업로드와 AI 챗봇의 연결이 명확한가? | 적합 | 업로드한 자료와 수정된 추출 정보를 AI 챗봇이 주간 계획 생성의 근거로 사용한다고 명시했다. |
| PRD의 MVP 핵심 기능을 포함하는가? | 적합 | 자료 업로드, AI 정보 추출, Google Calendar 기반 주간 계획, 주간 계획 생성 요일/시간 설정, 자연어 요구사항, 챗봇 기반 계획 조정, 추천 근거를 포함했다. |
| MVP 제외 기능이 들어갔는가? | 적합 | 다중 캘린더, 모든 LMS 자동 연동, 전체 월간 세부 계획, 수치형 완료 확률, 공동 일정 관리, 사용자의 직접 계획 수정, 주간 계획 다중 생성 등은 제외했다. |
| 월간 일정이 MVP 범위를 넘지 않는가? | 적합 | 월간은 일정 확인용 캘린더로만 정의하고, 월간 세부 자동 계획 생성은 제외했다. |

## 9. 결론

이 IA는 기존 MVP 범위를 유지하면서 화면 구조를 `온보딩 -> Google Calendar 연동 -> 하단 네비게이션 3개 탭 + 전역 플로팅 AI 챗봇`으로 재정리했다. 기능은 학업 자료 업로드, AI 일정 추출, Google Calendar 기반 오늘/주간 계획 확인, 업로드 자료 기반 주 1회 주간 계획 생성, 주 3회 AI 챗봇 기반 계획 조정으로 제한했으며, MVP 범위를 벗어나는 장기 확장 기능은 포함하지 않았다.

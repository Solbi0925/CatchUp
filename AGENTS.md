# AGENTS.md

## Product

Catchup은 대학생이 강의계획서, LMS 공지, 과제, 시험, Google Calendar 일정을 한곳에서 정리하고 오늘/이번 주에 무엇부터 해야 할지 판단하도록 돕는 AI 학업 일정 매니저 앱 MVP이다.

## Source of truth

작업 전에는 반드시 아래 문서를 순서대로 확인한다.

1. `PRD.md`: MVP 범위, 기능, 제외 기능, 성공 기준
2. `README.md`: 프로젝트 소개, 팀 정보, 현재 문서 구조
3. `IA.md`: 페이지 구조와 정보 구조
4. `FLOW.md`: 사용자 흐름
5. `SCREEN_SPEC.md`: 화면별 상세 명세

문서와 다른 요구가 생기면 먼저 `PRD.md`를 기준으로 판단한다.

## Build and verify

현재 단계는 기획 문서 작성 단계이다.

- 새 작업 전 `PRD.md`와 `README.md`의 내용이 서로 충돌하지 않는지 확인한다.
- 새 문서를 만들면 제목, 목적, 포함 범위가 명확한지 확인한다.
- 기능을 제안할 때는 10주 MVP 안에서 구현 가능한지 확인한다.
- 주간 계획 생성은 사용자가 설정한 요일과 시간에 주 1회만 가능해야 한다.
- AI 챗봇을 통한 일일/주간 계획 조정 요청은 1주에 최대 3번만 가능해야 한다.
- 코드가 생긴 뒤에는 앱 실행 방법, 테스트 방법, 환경 변수 사용 여부를 문서에 남긴다.

## Guardrails

- 실제 학생 이름, 학번, 전화번호, 이메일, 성적, 시간표, 캘린더 등 개인정보를 사용하지 않는다.
- 테스트에는 가짜 학생 데이터와 샘플 일정만 사용한다.
- 실제 학교 LMS 계정, 학교 포털 계정, Google 계정의 민감한 정보를 저장하지 않는다.
- API 키, OAuth 토큰, 비밀번호는 문서나 코드에 직접 적지 않는다.
- Google Calendar 연동 테스트에는 실제 민감 일정이 아닌 가짜 일정 또는 익명화된 샘플 일정을 사용한다.
- 업로드 예시는 실제 학생 자료가 아니라 익명화된 샘플 자료만 사용한다.
- AI 추천은 참고용 계획으로 표현하고, 정확한 완료 확률이나 성과 예측처럼 보이게 쓰지 않는다.

## Scope control

- 모든 작업은 `PRD.md`의 1차 MVP 범위 안에서 진행한다.
- 자동 LMS 완전 연동, 완벽한 OCR, 장기 학습 성과 예측, 친구 공동 일정은 만들지 않는다.
- Google Calendar 외 다른 외부 캘린더 연동이나 개인 시간표 직접 연동은 만들지 않는다.
- 파일 업로드는 최초 구현에서 PDF 또는 이미지 중 하나의 형식만 지원하고, 여러 형식 동시 지원은 이후 확장으로 분리한다.
- 고정된 월간 계획은 만들지 않는다. 현재 상태를 기준으로 앞으로 4주 일정을 윈도우처럼 확인해 이번 주 계획에 반영한다.
- 주간 계획은 사용자가 설정한 요일과 시간에 주 1회만 생성한다.
- 한 주의 계획 범위는 월요일부터 일요일까지로 한다.
- 같은 주 안에서 이미 주간 계획이 생성되었으면 추가 생성은 막고, 필요한 변경은 AI 챗봇의 계획 조정 흐름으로 처리한다.
- AI 챗봇을 통한 일일/주간 계획 조정 요청은 1주에 최대 3번만 허용한다.
- 할 일 완료 체크 후 남은 일정이 자동으로 즉시 재조정되는 기능은 MVP에서 제외한다.
- 사용자가 오늘 할 일 또는 주간 계획 항목의 시간, 날짜, 우선순위, 내용을 직접 수정하는 기능은 초기 구현에서 제외한다.
- 계획 생성 시 사용자의 자연어 요구사항을 반영하되, 구현 범위가 커지면 복잡한 개인화는 이후 확장 기능으로 분리한다.
- 일정 요약 캘린더에서 점 색상으로 학업 일정과 개인 일정을 구분하는 기능은 초기 구현 이후 추가 검토 기능으로 둔다.
- 일정 요약 캘린더에서 확인 필요 일정을 일반 일정과 다르게 표시하는 기능은 초기 구현 이후 추가 검토 기능으로 둔다.
- 새 기능을 추가하고 싶으면 먼저 “MVP에 꼭 필요한가?”를 확인한다.
- 기능이 커지면 이번 MVP가 아니라 이후 확장 기능으로 분리한다.

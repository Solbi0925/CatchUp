# CatchUp 자료 업로드 화면 디자인 명세

## 1. 문서 목적과 적용 범위

이 문서는 팀에서 최종 확정한 자료 업로드 화면 목업을 개발 구현 기준으로 변환한 화면 전용 디자인 명세다. 개발자는 목업 이미지를 다시 열지 않아도 이 문서만으로 `Upload` 탭의 **추출 완료 자료가 1개 있는 정상 상태**를 최대한 동일하게 구현할 수 있어야 한다.

이 문서의 범위는 다음과 같다.

- 자료 업로드 화면의 레이아웃과 시각 규칙
- 목업에 보이는 텍스트와 컴포넌트
- 현재 화면 상태와 가능한 상호작용
- Upload 화면의 구현 체크리스트와 확인이 필요한 항목

## 2. 기준과 판정 방법

### 값의 신뢰도 표기

| 표기 | 의미 |
| --- | --- |
| `확정` | 목업에서 텍스트, 존재 여부, 순서, 정렬, 상태처럼 직접 확인할 수 있는 값 |
| `추정` | 이미지 픽셀 비율과 390px 모바일 기준을 이용해 환산한 구현 권장값 또는 렌더링 색상 근사값 |
| `확인 필요` | 정적 이미지에서 결정할 수 없거나 다른 화면과의 전역 일관성 확인이 필요한 값 |

숫자 뒤의 `px`는 CSS 픽셀 기준 권장값이다. 첨부 PNG 자체 크기 `762 × 1516px`는 `확정`이지만 기기 프레임까지 포함한 이미지이므로, 내부 앱 화면의 수치는 390px 모바일 기준과 목업 비율을 함께 사용한 `추정`이다. 렌더링 안티앨리어싱, 그림자, 기기 프레임의 원근감 때문에 색상과 경계에는 약 `±2px`의 오차를 허용한다.

## 4. 전체 구조

### 4.1 위에서 아래로의 계층

```text
[iOS 상단 Safe Area / 상태 표시]

[헤더]
  자료 업로드                              폴더 아이콘

[학업 자료 업로드 영역]
  클라우드 업로드 아이콘
  학업 자료 업로드
  PDF 또는 이미지 파일

[업로드된 자료 섹션]
  업로드된 자료
  [PDF 파일 카드]
  [추출 결과 확인 및 수정 카드]

[유동 여백]

[AI Mate 플로팅 버튼]

[고정 하단 탭바]
  Today                 Month                 Upload

[iOS 하단 Safe Area / 홈 인디케이터]
```

### 4.2 레이아웃 원칙

- 앱 본문은 아주 옅은 쿨 라벤더 계열 배경 위에 놓인다. `확정`
- 헤더, 업로드 영역, 섹션 제목, 두 카드는 같은 좌우 기준선에 맞춘다. `확정`
- 기준 뷰포트 390px에서 좌우 본문 패딩은 `20px`로 구현한다. `추정`
- 콘텐츠 유효 너비는 약 `350px`다. `추정`
- 헤더와 카드 콘텐츠는 좌측 정렬이며, 업로드 영역 내부만 가로·세로 중앙 정렬이다. `확정`
- 하단 탭바는 화면에 고정되고, 플로팅 버튼은 탭바 위 오른쪽에 고정된다. 정적 이미지와 공통 흐름을 함께 근거로 한 구현 규칙이다. `추정`
- 자료 수가 늘어나면 헤더 아래 본문만 세로 스크롤하고 탭바는 고정한다. `확인 필요`

## 9. 컴포넌트 상세

### 9.1 시스템 Safe Area와 상태바

- 목업은 iPhone Dynamic Island, 시간, 신호, Wi-Fi, 배터리, 홈 인디케이터가 포함된 기기 프레임 안의 앱 화면이다. `확정`
- 웹앱 구현 대상은 기기 프레임과 시스템 상태 아이콘이 아니라 그 안의 앱 콘텐츠다. `확정`
- 상단은 `env(safe-area-inset-top)`, 하단은 `env(safe-area-inset-bottom)`을 반영한다. `추정`
- 상태바 콘텐츠 색상은 어두운 모드이며 앱 배경은 밝다. `확정`
- Dynamic Island, 시스템 시계, 홈 인디케이터를 앱 DOM으로 직접 그리지 않는다. `추정`

### 9.2 헤더

- 한 행에 좌측 제목과 우측 폴더 외곽선 아이콘을 배치한다. `확정`
- 제목은 좌측 본문 기준선에 붙이고 세로 중앙 정렬한다. `확정`
- 폴더 아이콘은 약 `27 × 25px`, 터치 영역은 최소 `44 × 44px`로 둔다. `추정`
- 아이콘은 검정에 가까운 `2px` 내외 라인의 둥근 폴더 외곽선이다. 채우기는 없다. `추정`
- 우측 아이콘의 의미는 업로드 파일 목록 또는 파일 선택 진입으로 보이지만, 정확한 동작은 이미지에서 결정할 수 없다. `확인 필요`

## 11. 스크롤과 고정 영역

- 헤더는 현재 목업만으로 sticky 여부를 판단할 수 없다. 일반 본문과 함께 스크롤하는 방식으로 우선 구현하되 팀 확인이 필요하다. `확인 필요`

- 업로드 영역 안의 자체 스크롤, 가로 스크롤, pull-to-refresh는 없다. `확정` 또는 `확인 필요`가 섞이는 항목으로, 목업 기준 노출하지 않는다.

## 15. 확인 필요 사항

개발 시작 전 팀이 확인해야 하는 Upload 화면 항목이다.

1. AI Mate 캐릭터 원본 SVG 또는 고해상도 PNG 에셋
2. 헤더 폴더 아이콘의 정확한 동작
3. 파일 카드 전체 클릭 여부와 추출 결과 카드와의 목적 차이
4. 메타 날짜의 의미
5. 업로드 파일 최대 용량, 허용 이미지 MIME, 다중 업로드 여부
6. 정확한 폰트 family와 font-weight
7. 빈 상태·추출 중·확인 필요·실패 상태의 최종 시각 목업

기능상 PDF와 이미지 지원, 사용자의 추출 결과 직접 수정, 3개 하단 탭, AI Mate 진입은 10주 MVP 범위 안에 있다.

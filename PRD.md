# AI 학업 일정 매니저 앱 MVP PRD

## 1. 해결하려는 문제

여러 과목을 듣는 대학생은 강의계획서, LMS 공지, 과제, 시험, 개인 일정이 여러 채널에 흩어져 있어 중요한 학업 일정을 놓치거나 무엇부터 해야 할지 판단하기 어렵다.

## 2. 주요 사용자

- 여러 과목을 동시에 수강하는 대학생
- LMS 공지와 강의계획서를 자주 확인해야 하는 학생
- 과제, 시험, 개인 일정을 함께 관리해야 하는 학생
- 일정 관리가 어렵거나 마감 직전에 몰아서 처리하는 학생
- 오늘 해야 할 일과 이번 주 우선순위를 빠르게 알고 싶은 학생

## 3. 사용 상황

1. 사용자가 강의계획서, LMS 공지, 과제 명세서 등 학업 자료 파일을 업로드한다.
2. AI가 파일에서 마감일, 시험일, 제출 방식, 과제명, 준비물, 중요 일정 등을 추출한다.
3. 사용자는 AI가 추출한 정보를 확인하고 잘못된 내용을 직접 수정한다.
4. 사용자가 Google Calendar를 연결한다.
5. 사용자가 일일/주간 계획 생성 시 “일요일에는 쉴 시간을 많이 확보해줘”, “매일 15분 이상 영어 공부 시간을 넣어줘”처럼 자연어 요구사항을 입력할 수 있다.
6. AI가 Google Calendar 개인 일정, 학업 일정, 과제 난이도, 예상 소요시간, 마감일, 사용자 요구사항을 함께 고려해 일일/주간 계획을 생성한다.
7. AI는 고정된 월간 계획이 아니라 현재 상태를 기준으로 앞으로 4주 일정을 윈도우처럼 다시 확인하고, 그 정보를 바탕으로 이번 주 계획을 업데이트한다.
8. 사용자는 AI 챗봇에게 “오늘 할 일 줄여줘”, “과제 먼저 하고 싶어”, “시험 공부 시간을 늘려줘”처럼 세부 계획 수정을 요청할 수 있다.
9. AI는 추천 결과와 함께 왜 이 계획을 추천했는지 근거를 설명한다.

## 4. MVP 기능

### 4.1 학업 정보 추출 기능

사용자는 강의계획서, LMS 공지, 과제 명세서 등 학업 자료 파일을 업로드할 수 있다.

초기 구현에서는 PDF 또는 이미지 중 하나의 파일 형식만 선택해 업로드를 지원한다. 해당 형식의 추출 품질과 사용 흐름이 검증되면 이후 확장 기능으로 다른 파일 형식을 추가한다.

AI는 업로드된 파일에서 다음 정보를 추출한다.

- 과제명
- 마감일
- 시험일
- 제출 방식
- 준비물
- 중요 공지
- 수업 일정 정보

추출된 정보는 사용자가 확인하고 수정할 수 있어야 한다. OCR 또는 문서 인식이 실패한 항목은 “확인 필요” 상태로 표시한다.

### 4.2 Google Calendar 기반 학습 계획 생성 및 업데이트 기능

서비스는 사용자의 Google Calendar 개인 일정을 불러온다. 다른 외부 캘린더나 개인 시간표 연동은 MVP 범위에 포함하지 않는다.

AI는 다음 정보를 함께 고려해 오늘 또는 이번 주 학습 계획을 생성한다.

- Google Calendar 개인 일정
- 과제 마감일
- 시험일
- 과제 난이도
- 예상 소요시간
- 사용자가 직접 수정한 우선순위
- 계획 생성 시 사용자가 입력한 자연어 요구사항

계획 생성 화면에는 일일/주간 계획 생성 버튼과 함께 사용자의 요구사항을 자연어로 입력하는 영역을 제공한다.

예시 입력:

- “일요일에는 좀 쉴 수 있는 시간을 많이 확보해주는 계획으로 짜줘.”
- “매일 15분 이상의 영어 공부 시간을 확보해줘.”
- “이번 주는 팀플 회의가 많으니 과제는 짧게 나눠서 배치해줘.”

MVP에서는 전체 월간 계획을 세부 단위까지 자동 생성하지 않는다. 대신 계획을 생성하거나 업데이트할 때마다 현재 상태를 기준으로 앞으로 4주 일정을 윈도우처럼 확인한다. 2~4주 뒤의 시험, 마감일, 제출일은 오늘 또는 이번 주에 미리 시작해야 할 준비 작업을 판단하는 참고 정보로 사용한다.

주간 계획을 다시 생성할 때는 기존 계획을 완전히 폐기하지 않고 현재 상태를 기준으로 업데이트한다.

- 완료한 일은 제거한다.
- 못 끝낸 일은 다시 배치한다.
- 사용자가 직접 수정한 일정은 최대한 유지한다.
- 새로 들어온 과제, 시험, 개인 일정은 반영한다.
- 우선순위가 바뀐 경우에만 일부 일정을 조정한다.

사용자는 생성된 일정의 시간, 날짜, 우선순위, 할 일 내용을 직접 수정할 수 있다. 사용자가 설정한 시간에 주 1회 AI가 이번 주 학습 계획을 생성하거나 업데이트한다.

### 4.3 AI 챗봇 기반 계획 조정 및 추천 근거 설명 기능

사용자는 AI 챗봇에게 세부 계획 조정을 요청할 수 있다.

예시 요청:

- “오늘 할 일 줄여줘.”
- “과제 먼저 하고 싶어.”
- “시험 공부 시간을 늘려줘.”
- “이번 주 개인 일정이 많으니 학습 계획을 다시 짜줘.”

AI는 강의계획서, 과제 명세서, Google Calendar 개인 일정, 사용자의 요청사항을 반영해 계획을 수정한다.

캘린더 또는 할 일 계획을 처음 제시할 때는 추천 근거를 함께 제공한다. 추천 이유에는 다음 요소를 포함한다.

- 마감일까지 남은 기간
- 예상 소요시간
- 중요도
- Google Calendar 개인 일정과의 충돌 여부
- 시험과의 연관성

추천 근거 예시:

- “예상 소요시간이 6시간이고 시험 범위와 연결되어 있어 오늘 먼저 시작하는 것을 추천합니다.”
- “마감일까지 남은 기간이 짧고 개인 일정과 충돌 가능성이 있어 우선순위를 높였습니다.”
- “오늘 시작하지 않으면 마감 전 완료 위험도가 높아질 수 있습니다.”

MVP에서는 실제 예측 모델 없이 “완료 확률 38%”처럼 정확한 수치형 예측을 제공하지 않는다. 대신 “완료 위험도 높음”, “우선순위 높음”처럼 정성적 표현을 사용한다.

## 5. 제외 기능

이번 해커톤 MVP에서는 아래 기능을 만들지 않는다.

- 모든 LMS와 학교 시스템의 완전 자동 연동
- 모든 파일 형식에 대한 완벽한 OCR/문서 인식
- 초기 선택 형식 외 PDF와 이미지 등 여러 파일 형식을 동시에 지원하는 기능
- 장기 학습 성과 예측 모델
- 정확한 완료 확률 계산
- 친구와의 공동 일정 관리
- 학교 행정 시스템과의 직접 연동
- Google Calendar 외 다른 외부 캘린더 연동
- 개인 시간표 직접 연동
- 할 일 완료 체크 후 남은 일정이 자동으로 즉시 재조정되는 기능
- 전체 월간 계획을 세부 단위까지 자동 생성하는 기능
- 여러 학교의 공지와 행사를 자동으로 수집하는 통합 플랫폼
- 장학금, 근로, 대외활동 개인화 추천 기능

## 6. 성공 기준

### 정량 기준

- 사용자가 강의계획서 또는 LMS 공지 파일을 업로드했을 때 핵심 일정 정보가 추출된다.
- 사용자가 Google Calendar 정보를 바탕으로 하루/주간 계획을 생성할 수 있다.
- 사용자가 계획 생성 시 자연어 요구사항을 입력하면 계획에 반영된다.
- 주간 계획 생성 시 앞으로 4주 안의 시험, 마감일, 제출일이 이번 주 계획에 참고 정보로 반영된다.
- MVP 사용 후 사용자가 오늘 해야 할 일을 1분 이내에 파악할 수 있다.
- 테스트 사용자 5명 이상이 “무엇부터 해야 할지 판단하는 데 도움이 된다”고 응답한다.

### 정성 기준

- 사용자가 AI 추천 이유를 보고 왜 해당 일정이 우선인지 이해할 수 있다.
- 사용자는 원문 공지나 강의계획서를 다시 읽지 않아도 마감일, 제출 방식, 다음 행동을 파악할 수 있다.
- 사용자는 계획을 직접 수정한 내용이 다음 계획 업데이트에서도 최대한 유지된다고 느낀다.
- 사용자는 이 서비스가 단순 캘린더가 아니라 학업 일정을 행동 계획으로 바꿔준다고 느낀다.

## 7. 10주 MVP 범위

### 1~2주차: 문제 검증 및 샘플 데이터 수집

- 강의계획서, LMS 공지, 과제 명세서 샘플 수집
- 추출해야 할 필드 확정
- 업로드, 추출 결과 확인, 계획 생성, 챗봇 수정 흐름 정의

### 3~4주차: 파일 업로드 및 학업 정보 추출

- PDF 또는 이미지 중 하나의 파일 형식 업로드 구현
- OCR 또는 문서 텍스트 추출 적용
- 과제명, 마감일, 시험일, 제출 방식 추출
- 사용자가 추출 결과를 수정하는 화면 구현

### 5~6주차: 개인 일정 연결 및 계획 생성

- Google Calendar 연동 기능 구현
- 현재 상태 기준 앞으로 4주 일정을 참고하는 오늘/이번 주 계획 생성 로직 구현
- 계획 생성 시 자연어 요구사항 입력 기능 구현
- 과제 난이도와 예상 소요시간 입력 기능 구현

### 7~8주차: 일정 수정, 완료 체크, 주간 계획 업데이트

- 사용자가 일정의 시간, 날짜, 우선순위, 내용을 수정하는 기능 구현
- 완료 체크 기능 구현
- 완료한 일 제거, 미완료 항목 재배치, 사용자 수정 일정 유지 규칙 구현
- 주 1회 학습 계획 생성 및 업데이트 기능 구현

### 9주차: AI 챗봇 및 추천 근거

- 챗봇 기반 계획 수정 요청 처리
- 추천 근거 문장 생성
- 수치형 확률 대신 정성적 우선순위와 위험도 표현 적용

### 10주차: 테스트 및 발표 준비

- 테스트 사용자 5명 이상으로 사용성 검증
- 추출 실패 사례와 계획 생성 오류 보완
- 데모 시나리오와 성공 기준 결과 정리

## 8. 다음 Codex 작업 요청문

이 PRD를 바탕으로 10주 안에 구현 가능한 앱 MVP의 기술 구현 계획을 작성해줘.

반드시 포함할 내용:

- 화면 구성: 파일 업로드, 추출 결과 확인/수정, 오늘/이번 주 계획, 계획 생성 요구사항 입력, AI 챗봇
- 데이터 모델: 사용자 일정, 학업 일정, 과제, 시험, 계획 항목, 추천 근거
- AI 프롬프트 초안: 정보 추출, 4주 윈도우 기반 계획 생성, 주간 계획 업데이트, 추천 근거 설명
- Google Calendar 연결 방식
- 10주 개발 일정
- 가장 먼저 구현할 작업 순서

범위는 모든 LMS 자동 연동이나 장기 예측 모델 없이, 업로드한 학업 자료와 Google Calendar 개인 일정을 바탕으로 오늘/이번 주 학습 계획을 생성하고 수정하는 앱 MVP로 제한해줘.
